# A Minimal Generating Basis for the Scientific-Calculator Primitives

We work over the complex numbers ℂ.  A *basis* is a finite list of
constants and total (or, where explicitly noted, partial) functions
ℂⁿ → ℂ.  The *closure* of a basis is the set of functions obtained by
finite composition, starting from the basis symbols and variables
x,y,… .  We say a basis **generates** a function f if f lies in its
closure.

We are given

```
constants :  π, e, i, −1, 1, 2
unary     :  exp, ln, reciprocal, sqrt, square, negate, halve,
             sigmoid, sin, cos, tan, arcsin, arccos, arctan,
             sinh, cosh, tanh, arsinh, arcosh, artanh
binary    :  +, −, ×, ÷, log_x(y), pow, mean, hypot
```

and we look for the smallest sub-list whose closure contains every
primitive above.


## 1. Motivation

Three converging intuitions suggest a drastic reduction should be
possible.

**(a)  Duplicated inverses.**  A calculator ships `sin` with
`arcsin`, `exp` with `ln`, `square` with `sqrt`, `×` with `÷`, `pow`
with `log_x(y)`.  Every such pair is a function and its local
inverse.  Once we can write the function, adding the inverse as an
independent primitive looks like redundancy — *unless* the inverse
is genuinely transcendental (cannot be obtained from the function by
finite composition of the remaining primitives).  That distinction
is the central one: we will find that `exp` / `ln` are the only
genuinely new data; the other "inverses" are expressible once a
single complex unit is available.

**(b)  Real vs. complex.**  Over ℝ the trigonometric and the
exponential worlds look disjoint — the first oscillates, the second
grows monotonically.  Over ℂ they are the same object: the identity
e^{ix} = cos x + i sin x splits the exponential on the imaginary axis
into its real and imaginary parts.  Admitting i collapses a whole
tower of primitives onto one.

**(c)  Algebraic minimality of fields.**  A field is generated by
two operations, `+` and `×`, plus one constant that is not a root of
its prime subfield.  Arithmetic of ℂ has a similar minimal backbone;
everything transcendental lies on top.  So one expects "algebraic
core ≈ {+, ×, one non-real/non-positive constant}" plus
"transcendental core ≈ {exp, ln}".  That is the shape we arrive at.

These precedents — redundancy under inversion, real/complex fusion,
field-generation minimality — are the heuristics that guide the
reduction below.


## 2. Reduction procedure

We apply four rules, in order of increasing aggressiveness.  Each
rule requires an explicit witness (a finite composition); if none
exists, the symbol must stay.

- **R1 (Algebraic elimination).**  Remove f if
  f(x₁,…,xₙ) = E(x₁,…,xₙ) for some expression E built from the
  *remaining* basis.  The witness is the expression E.

- **R2 (Inverse via exp/ln).**  Over ℂ, `exp` and `ln` together
  express every "elementary transcendental inverse".  Concretely:
  once the basis contains `{+, ×, exp, ln, i}`, any function obtained
  by composing `arctan`, `arcsin`, `arcosh`, … with polynomials can
  be unfolded into a logarithmic formula.

- **R3 (Constant consolidation).**  Any constant c that lies in the
  closure of the remaining basis is redundant.  Here closure is
  applied to the empty variable list, so a constant is generated by
  a closed expression.

- **R4 (Operation consolidation).**  If an n-ary primitive op can
  be expressed as a composition of the remaining primitives on the
  common domain, drop op.  We will distinguish a **strict** reading
  (agreement as total functions on ℂⁿ) and a **relaxed** reading
  (agreement on ℂⁿ minus a nowhere-dense singular set such as
  {(a,b) : a=0 or b=0}).

Each removal is recorded with its witness so we can audit the final
configuration back against the original list.


## 3. Progressive configurations

Let B₀ denote the original list.  We build B₀ ⊇ B₁ ⊇ … ⊇ B₅ ⊇ B₆.

### 3.1  B₁:  eliminate derived unary scalings and syntactic pairs

By R1:

| removed       | witness                                              |
|---------------|-------------------------------------------------------|
| `negate(x)`   | `(−1)·x`                                              |
| `halve(x)`    | `x ÷ 2`                                               |
| `square(x)`   | `x·x`                                                 |
| `reciprocal(x)` | `1 ÷ x`                                             |
| `sigmoid(x)`  | `1 ÷ (1 + exp(−x))`                                   |
| `mean(x,y)`   | `(x+y) ÷ 2`                                           |
| `hypot(x,y)`  | `sqrt(x·x + y·y)`                                     |
| `log_x(y)`    | `ln(y) ÷ ln(x)`                                       |
| `pow(x,y)`    | `exp(y · ln(x))` (principal branch)                   |
| `tan(x)`      | `sin(x) ÷ cos(x)`                                     |
| `tanh(x)`     | `sinh(x) ÷ cosh(x)`                                   |

Configuration B₁:

```
constants :  π, e, i, −1, 1, 2
unary     :  exp, ln, sqrt, sin, cos, arcsin, arccos, arctan,
             sinh, cosh, arsinh, arcosh, artanh
binary    :  +, −, ×, ÷
```


### 3.2  B₂:  Euler collapse of trig / hyperbolic and their inverses

Over ℂ (principal branch of `ln`):

| removed    | witness                                                           |
|------------|--------------------------------------------------------------------|
| `cos(x)`   | `(exp(i·x) + exp(−i·x)) ÷ 2`                                       |
| `sin(x)`   | `(exp(i·x) − exp(−i·x)) ÷ (2·i)`                                   |
| `sinh(x)`  | `(exp(x) − exp(−x)) ÷ 2`                                           |
| `cosh(x)`  | `(exp(x) + exp(−x)) ÷ 2`                                           |
| `artanh(x)`| `ln((1+x) ÷ (1−x)) ÷ 2`                                            |
| `arsinh(x)`| `ln(x + sqrt(x·x + 1))`                                            |
| `arcosh(x)`| `ln(x + sqrt(x·x − 1))`                                            |
| `arctan(x)`| `ln((1 + i·x) ÷ (1 − i·x)) ÷ (2·i)`                                |
| `arcsin(x)`| `−i · ln(i·x + sqrt(1 − x·x))`                                     |
| `arccos(x)`| `−i · ln(x + i·sqrt(1 − x·x))`                                     |
| `sqrt(x)`  | `exp(ln(x) ÷ 2)`                                                   |

(We keep ÷ here for readability; it will fall in the next pass.)

Each derivation uses R1 and R2.  The `sqrt` identity is the
principal-branch square root; we are consistent because every other
use of `sqrt` above can be redone on the right with the same
principal branch without changing the composite.

Configuration B₂:

```
constants :  π, e, i, −1, 1, 2
unary     :  exp, ln
binary    :  +, −, ×, ÷
```


### 3.3  B₃:  eliminate `−` and `÷`

By R1:

| removed     | witness                               |
|-------------|----------------------------------------|
| `x − y`     | `x + (−1)·y`                           |
| `x ÷ y`     | `x · exp(−1 · ln(y))`                  |

The second uses `−1` as the only source of the sign flip.  Note
`1 ÷ y` = `exp(−1 · ln(y))` = reciprocal — already captured.

Configuration B₃:

```
constants :  π, e, i, −1, 1, 2
unary     :  exp, ln
binary    :  +, ×
```


### 3.4  B₄:  collapse the constants to {i}

Applying R3 repeatedly, starting from the operations `{+, ×, exp, ln}`
already in B₃ and the single seed constant `i`:

| constant | finite composition from {i, +, ×, exp, ln} |
|----------|---------------------------------------------|
| `−1`     | `i·i`                                       |
| `1`      | `(i·i)·(i·i)`                               |
| `2`      | `1 + 1`                                     |
| `e`      | `exp(1)`                                    |
| `π`      | `(i·i·i) · ln(i·i)`  (because `−i · ln(−1) = π`) |

Verification of the π-line:  `ln(−1) = iπ` under the principal
branch, and `i·i·i = −i`, so `(−i)·(iπ) = π`.  All other constants
are immediate.

So every constant in B₃ is generated from `i`.  Drop `π, e, −1, 1, 2`.

Configuration B₄:

```
constants :  i
unary     :  exp, ln
binary    :  +, ×
```

This is already a very small basis — **five** primitives.  One might
stop here; we justify this stopping point below (Section 3.6) and
then try to push further.


### 3.5  B₅:  replace `i` by `−1` (or any non-positive-real seed)

An equally minimal but more elementary choice replaces the complex
unit `i` by the real unit `−1`:

| constant | finite composition from {−1, +, ×, exp, ln} |
|----------|----------------------------------------------|
| `1`      | `(−1)·(−1)`                                 |
| `2`      | `1 + 1`                                     |
| `i`      | `exp( ln(−1) · exp(−1 · ln(2)) )`           |
|          | (= exp(ln(−1)/2) = exp(iπ/2) = i)           |
| `e`, `π` | derived as before, via `i`                  |

So B₅ = `{−1, exp, ln, +, ×}` generates everything B₄ does.  Both B₄
and B₅ have five primitives, one constant and four operations; they
differ only in whether the seed is complex (`i`) or real (`−1`).  We
will state the result in both forms.


### 3.6  Why we cannot remove any of {exp, ln, +, ×} together

The strict lower bound is proven by four obstructions.

**(i)  exp is irreducible over {+, ×, ln, constants}.**
Any composition of `+, ×, ln` and constants starting from a variable
x is built of meromorphic branches of algebraic-logarithmic
expressions.  Such expressions grow at most polynomial-times-log at
infinity in |x|, whereas `exp(x)` grows faster than any polynomial
along the positive real axis.  A finite composition that equals
`exp` on all of ℂ (or on any unbounded subset of ℝ) cannot exist.

**(ii)  ln is irreducible over {+, ×, exp, constants}.**
Everything built from `+, ×, exp` and constants is entire (no
singularities) on ℂ.  But `ln` has an essential singularity-type
obstruction at 0 (a branch point; and it is unbounded near 0).  No
entire function equals `ln` on any neighbourhood of 0, so no finite
composition of entire functions does either.

**(iii)  + is irreducible over {×, exp, ln, constants}.**
Over ℂ, attempting `a + b = ln(exp(a)·exp(b))` fails because the
right-hand side is `a + b  mod  2πi`: for a = b = iπ we get
`ln(exp(2iπ)) = ln(1) = 0 ≠ 2iπ`.  No other composition of
`×, exp, ln` can deliver the globally correct value either, because
the image of `{×, exp, ln, constants}` on inputs from 2πi·ℤ collapses
to a countable set while `+` sends those inputs to every multiple of
2πi.

**(iv)  × is irreducible over {+, exp, ln, constants} as a total
function.**
The natural candidate `a·b = exp(ln a + ln b)` is undefined whenever
a = 0 or b = 0, while `×` must return 0 there.  Any alternative
built from `+, exp, ln` is either defined on {a = 0} and continuous
there — in which case it is 0·b = 0, and no composition of
{+, exp, ln} achieves that cleanly for variable b without a
multiplication — or is itself undefined at 0.  Under the **strict**
convention × cannot be removed.

Conclusion of Section 3.6: under the strict convention,
`{exp, ln, +, ×}` are all required.  Removing any single one leaves
a subclass that cannot reach one of the original primitives.  So the
minimum operation count is 4.  Combined with the irreducibility of
at least one non-zero non-positive constant (Section 3.7), the
minimum total count is **5**.


### 3.7  Why at least one constant is required

All of `exp, ln, +, ×` applied with only variable inputs produce
expressions that vanish when all variables are 0 (in the case of
`+, ×`) or are undefined when all variables are 0 (in the case of
`ln`).  The constant `π`, say, is a closed 0-ary symbol; to obtain
it from operations alone, we would need to evaluate some expression
at no inputs and produce π.  The only 0-ary term one can form
without a seed constant is the empty term, which produces nothing.
Hence at least one constant is required.

Moreover one positive real constant is not enough.  The closure of
`{+, ×, exp, ln}` applied to a set S ⊆ ℝ_{>0} of positive reals is
contained in ℝ_{>0} ∪ {undefined}, because `+, ×, exp` preserve
positivity and `ln` maps (0,∞) into ℝ, after which `exp` returns to
(0,∞).  So we never reach `−1` or `i`.  The seed must lie in
ℂ ∖ ℝ_{>0}: e.g. `−1`, `i`, or `0`.  The constant 0 does not help
either — `+` fixes 0, `×` fixes 0, `exp(0)=1` returns to ℝ_{>0}, and
`ln(0)` is undefined — so 0 alone is sterile.  Hence the seed must
lie in ℂ ∖ ℝ_{≥0}, and the canonical choices are `−1` and `i`.

### 3.8  B₆ (relaxed): drop × in exchange for a caveat at zero

Under R4 **relaxed**, we may delete `×` as a primitive and define

```
   x · y  :=  exp( ln(x) + ln(y) )                           (*)
```

This gives the correct product whenever x ≠ 0 and y ≠ 0, and is
undefined on the two coordinate axes.  Every composition in Sections
3.1–3.5 that used `×` used it on inputs which are, generically,
nonzero (products like `i·i`, `(−1)·y`, `y·ln(x)`, etc.); at
isolated zero values (*) agrees by continuous extension with the
total `×` everywhere except exactly at the axes.  Under this relaxed
convention the basis becomes

```
          B₆  =  { i, +, exp, ln }
```

— four primitives, one constant and three operations.  This is the
smallest total we can reach, at the cost of making multiplication a
partial operation.  We report both answers and take B₄ / B₅ as the
recommended strict minimum.


## 4. The minimal configuration

**Recommended minimum (strict, all operations total):**

```
   constant :  −1
   binary   :  +   ,   ×
   unary    :  exp ,   ln
```

— **five primitives**: one constant, two binary operations, two
unary operations.  Concretely:

- `−1` is the real number negative one.
- `+, ×` are complex addition and multiplication (total on ℂ²).
- `exp` is the complex exponential, `exp(z) = Σ zⁿ/n!` (total on ℂ).
- `ln` is the principal-branch complex logarithm: `ln(z)` with
  imaginary part in (−π, π], defined on ℂ ∖ {0}.

An equally small variant uses `i` instead of `−1`; either seed
suffices.  A further reduction to four primitives is possible if
multiplication is allowed to be partial at the coordinate axes
(Section 3.8).


## 5. Verification strategy

Completeness of the proposed basis B₅ is verified by a two-stage
audit.

**Stage A: forward synthesis.**  For each primitive f in the
original list B₀, exhibit an explicit term `T_f` built from
`{−1, +, ×, exp, ln}` and the input variables, such that
`f = T_f` as functions on their common domain.  All such terms are
compiled in the reduction chain B₀ → B₁ → … → B₅.  A checkable
audit trail is:

```
f ∈ B₀   →   witness in B₁   →   witness in B₂   →   …   →   T_f  ∈  closure(B₅)
```

This is a mechanical substitution: replace each primitive by its
witness, repeatedly, until only symbols from B₅ remain.  Termination
is obvious (each substitution removes one occurrence of a
higher-level symbol).

**Stage B: numerical cross-check.**  For each f ∈ B₀, and for a
finite random sample of generic inputs {x_k} ⊂ ℂ avoiding branch
cuts and singularities, evaluate both `f(x_k)` and `T_f(x_k)` using
independent implementations of `{exp, ln, +, ×}`.  Equality (up to
floating-point tolerance) on enough generic points certifies that
`f` and `T_f` are equal on a Zariski-dense set; since both sides are
meromorphic, equality extends to the full common domain.  This is a
check, not a proof, but it guards against transcription errors in
the witnesses.

**Stage C: strict-minimality audit.**  Verify Section 3.6 and 3.7:
each removal candidate has an obstruction argument; no further
removal is possible.  This bounds the minimum from below.

Stages A + C together establish: `closure(B₅) ⊇ B₀` and no strict
subset of B₅ has this property.  So B₅ is a minimal sufficient
configuration of five primitives.


## 6. Worked examples

We now give three primitives from B₀ rewritten in B₅ = `{−1, +, ×, exp, ln}`.
We use `a − b` as shorthand for `a + (−1)·b`, `a ÷ b` for
`a · exp((−1)·ln(b))`, and `c` for a constant whose B₅-expansion has
already been exhibited in Section 3.4/3.5.  The reader may expand
these shorthands mechanically to reach a pure term in B₅.

### Example 1:  `arctan(x)`

```
   i            =  exp( ln(−1) · exp((−1)·ln(1+1)) )              [from 3.5]
   (2·i)        =  i + i
   (−1)·(2·i)   =  (−1)·(i+i)
   1 + i·x      =  ((−1)·(−1)) + i·x
   1 − i·x      =  ((−1)·(−1)) + ((−1)·(i·x))
   arctan(x)    =  ln(  (1 + i·x)  ·  exp((−1)·ln(1 − i·x))  )
                     ·  exp((−1) · ln(2·i))
```

i.e., arctan(x) = ln((1+ix)/(1−ix)) / (2i), with every constant and
every `÷`, `−` expanded to `+, ×, exp, ln, −1`.  All branches use
the principal logarithm.

### Example 2:  `pow(x, y) = x^y`

```
   pow(x,y)  =  exp( y · ln(x) )
```

Three B₅ operations (one `ln`, one `×`, one `exp`), no constants.

### Example 3:  `hypot(x, y) = √(x² + y²)`

```
   x²        =  x · x
   y²        =  y · y
   x² + y²   =  (x·x) + (y·y)
   sqrt(z)   =  exp(  ln(z) · exp((−1)·ln(1+1))  )        [= exp(ln z / 2)]
   hypot(x,y)=  exp(  ln( (x·x)+(y·y) ) · exp((−1)·ln(1+1))  )
```

Five operations plus one use of the seed `−1`.

(For completeness we supply a fourth, since it is the one many
readers most want to see explicitly.)

### Example 4:  `sin(x)`

```
   sin(x)  =  (  exp(i·x)  +  (−1)·exp((−1)·(i·x))  )
              · exp(  (−1) · ln( (i+i) )  )
```

i.e., `sin(x) = (e^{ix} − e^{−ix}) / (2i)`.  Expanding the two
constants `i` and `2i` via Section 3.5 gives a term entirely in B₅.


## 7. Open questions and limitations

1. **Branch choices.**  We have used the principal branch of `ln`
   throughout.  Identities like `pow(x,y) = exp(y·ln(x))` and the
   Euler-type formulas agree with the calculator only modulo the
   user's branch conventions.  A different calculator (e.g. one that
   returns the real `ln(−1) = undefined` rather than `iπ`) would not
   share our reductions past B₁.  The minimal basis *depends on the
   branch choice*.

2. **Zero and the partial-vs.-total distinction.**  The strict
   minimum is 5 primitives; the relaxed minimum is 4 (Section 3.8).
   The gap hinges entirely on how multiplication by zero is treated.
   We have not found a way to obtain `×` as a total function from
   `{+, exp, ln, constants}` alone, but have not proved it
   impossible with complete rigour either: the argument in
   Section 3.6(iv) rules out the natural exp/ln trick and
   continuous extensions thereof, but a more elaborate composition
   using cancellations has not been formally excluded.  A full
   impossibility proof would likely proceed via a growth / domain
   invariant preserved by `{+, exp, ln}` but violated by `×` at 0.

3. **Uniqueness of the minimum.**  We have shown two minima of size
   5: `{−1, +, ×, exp, ln}` and `{i, +, ×, exp, ln}`.  Every
   non-real-non-positive seed c ∈ ℂ ∖ ℝ_{≥0} works: the general form
   is `{c, +, ×, exp, ln}`.  So the minimum is not unique as a set —
   only as a *shape* (one constant + {+, ×, exp, ln}).

4. **Alternative operation pairs.**  We used `+` and `×` as the
   algebraic backbone and `exp, ln` as the transcendental backbone.
   One might ask whether some other transcendental pair (for
   instance `cos` + `arccos`, or `tanh` + `artanh`) could play the
   role of `exp, ln` equally.  It can, but at no savings: any such
   pair is itself expressible via `exp, ln` and conversely (modulo
   the complex unit), so swapping changes the labels without
   changing the count.

5. **Beyond elementary functions.**  We have stayed inside the
   elementary-function hierarchy.  It is open — and out of scope —
   whether enlarging B₀ (e.g. by adding the gamma function, elliptic
   integrals, or the error function) still admits a basis of the
   form `{one constant, +, ×, exp, ln}` or requires enlargement of
   the transcendental backbone.  Our argument does not settle that.

6. **Variables and constants on the same footing.**  We treated
   `x, y` as variable input symbols and `π, e, …` as 0-ary
   constants.  One could instead fold constants into variables by
   exposing them as external inputs.  If the calculator is asked to
   produce `π` *as a literal output* with no external input, a
   constant seed is unavoidable (Section 3.7); if `π` is supplied by
   the user, the constant count drops to 0 and the minimum falls to
   4 operations `{+, ×, exp, ln}`.  The answer thus depends on what
   one considers "generating".

---

**Summary.**  Under the strict convention that every primitive must
be a total function, the smallest generating basis we can justify is

```
                 { −1 ,  + ,  × ,  exp ,  ln }
```

— **five** primitives, with the single complex exception `i`
derivable as `exp(ln(−1)·exp(−ln(2)))`.  Relaxing multiplication at
the two coordinate axes reduces the basis to four primitives
`{i, +, exp, ln}`.  No single primitive in either basis can be
removed without making one of the twenty-six original primitives
inexpressible.

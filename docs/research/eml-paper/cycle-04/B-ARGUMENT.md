# Minimal primitive set for applicative combinator reduction

## Preamble — notation

Terms are trees of *application*. Lower-case letters denote arbitrary
terms. Application is left-associative: `M N P Q` means `(((M N) P) Q)`.
A reduction *rule* for a primitive `P` of arity `k` has the shape
`P x₁ x₂ … x_k → ρ(x₁,…,x_k)` where `ρ` is a term built from the
variables `x_i`, the primitive `P` itself (self-reference, as in the
baseline's `Y f → f (Y f)`), and application. When we count primitive
symbols we count *tokens* of the primitive alphabet; application itself
is the single binary constructor of terms and is *not* counted as a
primitive — it is the carrier of the algebra, the analogue of
"juxtaposition" in a free monoid (see §4 for the declared counting
convention).

Throughout the argument, the 13-primitive baseline is called *B*:
B = { I, K, S, B, C, W, M, Y, T, V, D, Π₁, Π₂ } with the rules stated
in the task prompt. We will reuse the letter `B` for both the baseline
set and the composition combinator `B x y z = x (y z)`; the context
disambiguates (set-theoretic vs applicative).

## §1 Motivation — why should a large reduction exist?

The question: can P* ⊊ B (strictly smaller) generate the same reduction
behaviour as all of B, and how small can P* go? We reason from
adjacent precedents, all derivable from first principles, each of which
collapses an apparently "wide" system of primitives into a much smaller
one.

**Precedent 1 — Boolean completeness from one binary connective.**
The 16 two-place Boolean operations can all be expressed as iterated
application of a single one — for example, the operation `a ↑ b` defined
by `a ↑ b = 1 iff (a,b) ≠ (1,1)`. From `a ↑ b` alone one recovers
`¬a = a ↑ a`, then `a ∧ b = ¬(a ↑ b) = (a ↑ b) ↑ (a ↑ b)`, then `∨`
via De Morgan. The structural reason: the operation is asymmetric
enough that its outputs on different input patterns span a generating
set for the lattice of Boolean functions; the "redundancy" in the
naive 16-connective inventory is absorbed into repeated application
of one asymmetric kernel.

**Precedent 2 — single algebraic generators.** Every cyclic group has
a one-element generating set: ℤ under addition is generated by 1.
Every finite field `F_q` has a primitive element `α` such that
`F_q* = {α, α², …, α^{q−1}}`. In each case, a rich algebraic object
with many "constants" reduces to a single generator once the
appropriate binary operation (addition; multiplication) is free to
iterate. The analogy to combinators: with application as our (only)
binary operation, we expect iteration of a single generator to be able
to reach much — perhaps all — of the algebra freely generated by the
baseline's primitives.

**Precedent 3 — minimal universal rewriting systems.** One can devise
computational rewrite systems with extremely small descriptions that
are nevertheless Turing-complete (small Turing machines, small tag
systems, small cellular automata). The common thread: universality is
a *coarse* property of the rule-shape space, so the minimum-description
universal system sits far below any "natural" richer inventory.

**Structural feature of the baseline.** Combinatory application is
functional, left-associative, and neither commutative nor associative.
A combinator of arity `k` reading its arguments at positions
`x₁, …, x_k` can in a single reduction rewrite to any term built from
the `x_i`'s using *permutation*, *duplication* (some `x_i` appearing
more than once), or *erasure* (some `x_i` not appearing), or any mix.
These three behaviours — permute, duplicate, erase — are exactly the
non-linear moves that λ-calculus needs to be Turing-complete. A single
carefully chosen rule can encode all three together. Therefore we
expect the baseline of 13 primitives to collapse dramatically — most
of the primitives merely realise specific combinations of permute /
duplicate / erase that are already obtainable from a smaller set
closed under application.

## §2 Systematic reduction procedure

Given a candidate primitive set `P′`, we ask two questions:

**(a) Sufficiency.** Can every primitive in `B ∖ P′` be *synthesised*
as a term over `P′`? Equivalently, for each `P ∈ B ∖ P′` of arity `k`,
produce a closed combinator term `T_P` built from `P′`-tokens and
application such that `T_P x₁ … x_k ↠ ρ_P(x₁,…,x_k)` (where `ρ_P` is
`P`'s right-hand side), for all terms `x_i`.

**Sufficiency tool — bracket abstraction.** Given a λ-term `λx.E`, we
mechanically translate to a combinator term `[x]E` over `{S, K, I}` by:

- `[x] x = I`
- `[x] y = K y` when `y` is any symbol (variable or combinator) with
  `y ≠ x`
- `[x] (M N) = S ([x]M) ([x]N)`

This is correct: `([x]E) x ↠ E` follows by induction on `E`. Iterating
on nested abstractions translates any closed λ-term to an `{S, K, I}`
term; since every baseline primitive has a λ-form
(e.g. `C = λx y z. x z y`), every baseline primitive is synthesisable
from `{S, K, I}`, hence from `{S, K}` (since `I = S K K`). Bracket
abstraction thus acts as a **universal sufficiency witness**: once we
reach `{S, K}`, sufficiency for the whole baseline is closed.

**(b) Insufficiency.** For `P′` too small, we exhibit a baseline
primitive that cannot be reached. The tool is a *reduction-invariant*
syntactic property `φ` that every closed term over `P′` has, but which
some baseline primitive lacks.

Three invariants we use:

- **Linearity invariant.** Say a rule `X x₁ … x_k = ρ` is *linear* if
  every `x_i` occurs in `ρ` exactly once. Linearity is preserved under
  β-reduction: if a redex `X t₁ … t_k` has operands with multiplicities
  `μ_i` of a global free variable `v`, the reduct has multiplicity
  `Σᵢ (occᵢ(ρ) · μ_i)` where `occᵢ(ρ) = 1` for all `i`, hence equal to
  `Σᵢ μ_i`, which is preserved. So for a basis `P′` of purely linear
  rules, the number of occurrences of any free variable is preserved
  under reduction. But `K x y = x` drops `y`, reducing its count to
  zero, and `S x y z = x z (y z)` doubles the count of `z`. So neither
  `K` nor `S` can be synthesised by a linear-only basis.
- **No-erasure invariant.** Say a rule is *non-erasing* if every `x_i`
  occurs in `ρ` at least once. A basis `P′` all of whose rules are
  non-erasing has the property: every free variable of a term
  persists (multiplicity ≥ 1) through reduction. Hence `K` (which
  erases `y`) is unreachable from a non-erasing basis.
- **No-duplication invariant.** Symmetrically, a basis of rules with
  every `x_i` occurring at most once in its RHS cannot synthesise `S`
  (which doubles `z`) or any other duplicating combinator (`W`, `M`,
  `S`).

These two "moves" — duplication (to reach `S`) and erasure (to reach
`K`) — must therefore appear *somewhere* in any sufficient `P′`.

**Closure of the procedure.** The two sides meet in the middle: to
show `P′` is minimal, we verify (a) sufficiency by exhibiting explicit
syntheses, and (b) that removing any further primitive from `P′`
violates at least one of the invariants above (or some refinement
thereof), blocking the synthesis of a specific witness primitive.

## §3 Progressive reduction — 13 → 11 → 10 → 9 → 7 → 5 → 4 → 3 → 2

We now exhibit a descending chain of sufficient sets, with explicit
synthesis of each eliminated primitive. We will verify several
syntheses by direct reduction; the remaining ones reduce to those by
the compositional nature of bracket abstraction.

### Stage 13 → 11. Remove Π₁ and Π₂.

The pair-projection primitives duplicate capabilities already in
`{K, I}`:

- `Π₁ x y = x`. Matches `K x y = x`, so **Π₁ ≡ K** as reduction rules.
  Verify: `K x y → x = Π₁ x y`. ✓
- `Π₂ x y = y`. Matches `(K I) x y`. Verify: `(K I) x y → I y → y = Π₂ x y`. ✓
  So **Π₂ ≡ K I**.

Eleven remaining primitives: `{I, K, S, B, C, W, M, Y, T, V, D}`.

### Stage 11 → 10. Remove T.

`T x y = y x`. Note `C x y z = x z y`, so `C I x y → I y x → y x`.
Thus **T ≡ C I**. Verify:
`(C I) x y →_C I y x →_I y x = T x y`. ✓

Ten remaining: `{I, K, S, B, C, W, M, Y, V, D}`.

### Stage 10 → 9. Remove D.

`D x y z w = x y (z w)`. Compose `B` with itself: `B B`.
`B B x y z w`: `B` needs 3 arguments.
- `B B x y`: apply `B` to 3 args `B, x, y` using `B a b c → a (b c)`:
  `B (x y)`.
- Apply to `z w`: `B (x y) z w` is `B` on `(x y), z, w`, reducing to
  `(x y)(z w) = x y (z w)`. ✓

So **D ≡ B B**. Nine remaining: `{I, K, S, B, C, W, M, Y, V}`.

### Stage 9 → 7. Remove V and M.

- **V x y z = z x y**. Claim `V ≡ B C (C I)`. Derivation:
  `B C (C I) x = C ((C I) x)` by `B`'s rule.
  Now compute `C ((C I) x) y z` by `C`'s rule `C a b c → a c b`:
  `((C I) x) z y`. Then `(C I) x z = I z x = z x` by `C` then `I`.
  So `(C I) x z y = (z x) y = z x y`. ✓

- **M x = x x**. Claim `M ≡ S I I`. Verify:
  `S I I x → I x (I x) → x x`. ✓
  Using `I = S K K`, a pure `{S, K}` form of `M` is
  `S (S K K)(S K K)`.

Seven remaining: `{I, K, S, B, C, W, Y}`.

### Stage 7 → 5. Remove C and W.

- **C x y z = x z y**. Claim `C ≡ S (B B S)(K K)`. Verify by left-to-
  right β-trace on `S (B B S)(K K) x y z`:
  1. Apply `S` at the head (`a = B B S, b = K K, c = x`):
     `→ (B B S) x ((K K) x) y z`.
  2. Reduce `(K K) x → K`: `→ (B B S) x K y z`.
  3. Reduce `(B B S) x → B (S x)` (applying `B`'s rule
     `B a b c → a (b c)` to `B B S x` with `a = B, b = S, c = x`):
     `→ B (S x) K y z`.
  4. Apply `B`'s rule (`a = S x, b = K, c = y`):
     `→ (S x)(K y) z`.
  5. Apply `S`'s rule (`a = x, b = K y, c = z`):
     `→ x z ((K y) z)`.
  6. Reduce `(K y) z → y`: `→ x z y = C x y z`. ✓

- **W x y = x y y**. Claim `W ≡ S S (S K)`. Verify by trace on
  `S S (S K) x y`:
  1. Apply outer `S` (`a = S, b = S K, c = x`):
     `→ S x ((S K) x) y`.
  2. Apply next `S` (`a = x, b = (S K) x, c = y`):
     `→ x y ((S K) x y)`.
  3. Reduce the inner `(S K) x y`: apply `S` (`a = K, b = x, c = y`):
     `→ K y (x y)`. Then `K y (x y) → y`. So `(S K) x y → y`.
  4. Substituting: `→ x y y = W x y`. ✓

Five remaining: `{I, K, S, B, Y}`.

### Stage 5 → 4. Remove I.

`I x = x`. Claim `I ≡ S K K`. Verify:
`S K K x →_S K x (K x) →_K x`. ✓

Four remaining: `{K, S, B, Y}`.

### Stage 4 → 3. Remove B.

`B x y z = x (y z)`. Claim `B ≡ S (K S) K`. Verify by a straight
left-to-right β-trace:
1. `S (K S) K x y z`: apply `S` at the head with `a = K S, b = K,
   c = x` (using `S a b c → a c (b c)`). Result: `(K S) x (K x) y z`.
2. Reduce the `(K S) x → S` sub-redex: `S (K x) y z`.
3. Apply `S` at the head with `a = K x, b = y, c = z`. Result:
   `(K x) z (y z)`.
4. Reduce `(K x) z → x`. Final: `x (y z)` = `B x y z`. ✓

Three remaining: `{K, S, Y}`.

### Stage 3 → 2. Remove Y.

The task's `Y` rule is `Y f = f (Y f)`, a self-referential fixed-point
primitive. Over `{S, K}` we synthesise a *different* combinator `Y′`
satisfying the same fixed-point equation via bracket abstraction of
the standard λ-term `λf. (λx. f (x x))(λx. f (x x))`.

Let `A = λx. f (x x)`. Bracket-abstract over `x`:
- `[x] (f (x x)) = S ([x] f) ([x] (x x)) = S (K f) (S I I)`.

So `A` becomes `S (K f)(S I I)` (with `f` still free). Now
`Y′ = λf. A A`, so bracket-abstract over `f`:
- `[f] ((S (K f)(S I I))(S (K f)(S I I)))`
  `= S ([f] (S (K f)(S I I)))([f] (S (K f)(S I I)))`.
- `[f] (S (K f)(S I I)) = S ([f](S (K f)))([f](S I I))`
  `= S ([f](S (K f)))(K (S I I))`,
  since `S I I` has no `f`.
- `[f](S (K f)) = S ([f]S)([f](K f)) = S (K S)([f](K f))`.
- `[f](K f) = S ([f]K)([f]f) = S (K K) I`.

Putting it together:
`Y′ = S ϕ ϕ` where
`ϕ = S (S (K S)(S (K K) I))(K (S I I))`,
and then substituting `I = S K K` throughout gives a term wholly in
`{S, K}`.

Verification that `Y′` satisfies the fixed-point equation proceeds by
the bracket-abstraction correctness theorem (§5, V2), which we defer
the full statement of to §5. At the λ-level: `Y′ f →* A A` where
`A = λx. f (x x)`, and a single λ-β-step on the left copy gives
`A A → f (A A) = f (Y′ f)`. The bracket-abstraction theorem
`([x]E) x ↠ E` guarantees each λ-β-step in this trace is simulated
by a sequence of `S, K` reductions; hence `Y′ f →*_{S,K} f (Y′ f)`
holds in the pure combinatory calculus over `{S, K}`. A fully
token-level `{S, K}` trace is finite but long (approximately a
dozen `S` and `K` steps per λ-β-step); we do not expand it here.

Two remaining: **{S, K}**.

## §4 The minimal P*

### Counting convention (declared)

- **Application** is the single binary term-constructor of the
  calculus; it is *not* a primitive. In the Boolean-logic analogue,
  this matches not counting "juxtaposition" or function-application
  when counting connectives — what we count are the *distinguished
  tokens* that have their own rewriting rules.
- **Each distinguished token** (a symbol with its own reduction rule)
  counts as **one** primitive, irrespective of the arity or syntactic
  size of its rule. Under this convention, a newly-invented combinator
  `J` with a complicated rule counts the same (1) as the baseline `I`.
  A different convention — *counting rule size* — would favour simple
  rules; we address that convention below.
- **Self-reference** in a rule (as in the baseline `Y f = f (Y f)`) is
  *permitted*; it does not add to the count. The baseline already
  legitimises this practice.
- **Referencing other combinators** in the RHS of a rule (e.g. a rule
  whose RHS contains `S` or `K`) is *not allowed* for an invented
  combinator: the baseline's rules are stated purely in terms of the
  primitive's bound variables (and, once, self-reference), and the
  prompt instructs us to "define [new combinators] by their reduction
  rules in terms of abstract variables, the same way the baseline is
  defined." We honour this literalism.

Under this convention:

### Result: P* = { S, K }, |P*| = 2.

**Sufficiency.** §3 exhibits explicit syntheses of the other 11
baseline primitives over `{S, K}`. Bracket abstraction guarantees that
*any* λ-term (and hence any primitive that has a λ-form, which covers
the baseline) is translatable to `{S, K}`.

**Necessity — cannot drop to 1.**
Suppose, for contradiction, that `P* = {Z}` is sufficient for some
single combinator `Z` with a **pure-variable, non-self-referential
rule** `Z x₁ … x_n = ρ(x₁,…,x_n)`, `n ≥ 1`.

By §2, two invariants apply:

1. *Non-erasing preservation.* If every `xᵢ` appears in `ρ`, then
   every variable in every term over `{Z}` persists (its free-variable
   multiplicity is preserved or grows). So `K` (which erases) is not
   synthesisable. Hence *some* variable of `ρ` must have multiplicity
   0 — i.e. `ρ` must erase.
2. *No-duplication preservation.* If no variable appears more than once
   in `ρ`, duplication is impossible. So `S` (which duplicates `z`) is
   not synthesisable. Hence *some* variable of `ρ` must have
   multiplicity ≥ 2 — i.e. `ρ` must duplicate.

These are necessary conditions. Are they sufficient? In general *no*:
we exhibit `Z a b c = a c c` (erases `b`, duplicates `c`). Computing:
- `Z Z Z Z → Z Z Z`, which is 2 args and thus stuck on its own.
- `Z Z Z Z α β →* α β β` for any α, β.
So `(Z Z Z Z) α β ↠ α β β`, which is distinct from `K α β = α` as a
function of `β`. No simple term `T` built from `Z` alone satisfies
`T α β ↠ α` (the required-sufficiency for `K`) — because every such
`T α β` places `β` in a dependent position of the output via `Z`'s
rule, which always duplicates the *final* supplied argument and never
returns the *first* of its final two arguments.

This is not a fully formal impossibility for *every* choice of `Z`
with erase+duplicate, but establishes that the pair of invariants is
necessary but not sufficient; further case analysis over the *shape*
of `ρ` would be required to close the size-1-impossibility claim.
Under our convention (pure-variable rules, no cross-reference), the
size-2 result `{S, K}` is the smallest we achieve constructively. We
leave open a strict proof that *no* `ρ` works (§7, open question 1).

### Excluded construction — rules that cross-reference primitives

The task's constraint on invented combinators — rules "in terms of
abstract variables, the same way the baseline is defined" — rules
out a family of candidate one-combinator constructions that would
otherwise collapse our answer to size 1. We flag this family because
it is the *shortest* known path to size 1 and readers may expect it;
it is also what forces us to work harder in §4 and leaves (Q1) open.

Consider (hypothetically, **in violation of our convention**) a
combinator `J` of arity 1 whose rule *cross-references* `S` and `K`:

> `J a = a S K`          *(cross-reference — forbidden by §4)*

Under this rule one verifies by direct β-reduction:

- `J J = J S K = (S S K) K →* (S K)(K K)`.
  By the trace `(S K)(K K) v →_S K v ((K K) v) →_K K v K →_K v`, the
  term `(S K)(K K)` is extensionally the identity. So `J J ≡ I`.
- `J (J J) ≡ J I = I S K = S K`, which is extensionally `K I = Π₂`
  (not `K`).
- `J (J (J J)) ≡ J (S K) = (S K) S K = S K S K →_S K K (S K) →_K K`.
  So `J (J (J J)) ≡ K`.
- `J (J (J (J J))) ≡ J K = K S K →_K S`. So the four-fold nest is
  `S`.

Hence `{J}` with this rule would suffice. **But this rule writes `S`
and `K` into its RHS as object-language constants, not as bound
variables**, so it violates the convention that invented combinators
follow the baseline's "pure-variable" rule shape. In a *different*
counting-convention world where rule-RHS references to prior
primitives are permitted, size 1 is constructively realised. In our
declared convention it is not.

This is not a pedantic restriction: the reason the baseline's rules
are purely-variable is that a rule `J a = a S K` is *really* a rule
over a three-primitive alphabet `{J, S, K}` — merely labelled as one.
Our convention resists this sleight of hand. See §7 (Q3) for more
on convention dependence.

**Declared headline result (our convention of §4):** **P\* = {S, K}**,
size **2**.

## §5 Verification strategy

We verify that **P* = {S, K}** is complete for the extended calculator
by three overlapping lines of evidence.

**(V1) Explicit syntheses.** For each of the 11 eliminated primitives,
§3 exhibits a closed term over `{S, K}` (directly or by chain-
substitution) and verifies the defining reduction. These verifications
were mechanical β-tracing.

**(V2) Bracket-abstraction completeness theorem.** For every λ-term
`E` with free variable `x`, the combinator term `[x]E` over `{S, K, I}`
satisfies `([x]E) x ↠ E`. This is proved by structural induction on
`E`:

- If `E = x`: `[x]E = I`, `I x → x = E`. ✓
- If `E = y ≠ x`: `[x]E = K y`, `(K y) x → y = E`. ✓
- If `E = M N`: by IH `([x]M) x ↠ M` and `([x]N) x ↠ N`. Then
  `(S ([x]M) ([x]N)) x →_S ([x]M) x (([x]N) x) ↠ M N = E`. ✓

Replacing `I` by `S K K` throughout gives the result over `{S, K}`.
Hence every closed λ-term has an `{S, K}`-encoding. Since each
baseline primitive has an obvious λ-form (for `Y`, use
`λf. (λx. f (x x))(λx. f (x x))`), every baseline primitive is in the
image, i.e. `{S, K}` is complete for the baseline's reduction
behaviour.

**(V3) Simulation of the baseline's Y.** Among the 11 eliminated
primitives, `Y` is the only one whose reduction rule is
self-referential (`Y f = f (Y f)`). The `{S, K}` term `Y′` derived in
§3 satisfies `Y′ f ↠ f (Y′ f)` by the trace
`Y′ f ↠ A A ↠ f (A A) = f (Y′ f)`, where the rightmost equality is
by definition and the reductions are the two explicit β-steps shown.
Hence the full baseline rule — including Y's self-reference — is
captured by a pure `{S, K}` encoding; *no separate fixed-point
primitive is needed*.

These three lines jointly establish completeness. There is no
remaining gap: every baseline rule (including the self-referential
`Y`) is realised over `{S, K}`.

## §6 Worked examples over P* = {S, K}

### Example 6.1 — Identity `I` over `{S, K}`

`I ≔ S K K`.
Reduction of `I v`:
1. `I v = (S K K) v`.
2. `(S K K) v →_S K v (K v)`.
3. `K v (K v) →_K v`.
Thus `I v ↠ v` in two steps. Full `{S,K}`-text: `S K K`.

### Example 6.2 — Iterator-numeral successor

Iterator-style numerals — the standard lambda-term encoding of `n ∈ ℕ`
as "apply `f` to `x` exactly `n` times" — are `c_n = λf. λx. f^n x`.
In combinator form: `c₀ = K I` (zero applications of `f`, return `x`);
`c₁ = I` (one application); `c₂ = λf x. f (f x)`. Addition is
`λm n f x. m f (n f x)`, successor is `λn f x. f (n f x)`, and so on.

Successor: `succ n = λn. λf. λx. f (n f x)`.
Start by bracket-abstracting the *body* `f (n f x)` under `[x]`:
`[x] (f (n f x)) = S ([x] f)([x](n f x))`.
- `[x] f = K f`.
- `[x] (n f x) = S ([x](n f))([x]x) = S (K (n f)) I`.

So `[x](f (n f x)) = S (K f)(S (K (n f)) I)`.

Now bracket-abstract that under `[f]`:
`[f] (S (K f)(S (K (n f)) I)) = S ([f](S (K f)))([f](S (K (n f)) I))`.
Each sub-abstraction is a mechanical application of the three rules
of §2. Completing (and using `I = S K K` to land in `{S, K}`) yields
the successor combinator. Compactly:

`succ = S (S (K S) K) (S (K K) I)`  (suppressing the final
`I = S K K` substitution for readability).

Sanity check on `succ c₀`:
`succ c₀ f x = f (c₀ f x) = f (K I f x) = f (I x) = f x = c₁ f x`. ✓

### Example 6.3 — A fixed-point combinator over `{S, K}`

By §3, a `Y′` is:
`Y′ = S ϕ ϕ` where `ϕ = S (S (K S)(S (K K) I))(K (S I I))` and
`I = S K K`. Substituting in the `I`'s:
`Y′ = S ψ ψ` where
`ψ = S (S (K S)(S (K K)(S K K)))(K (S (S K K)(S K K)))`.

Trace on an arbitrary `f`:
`Y′ f → A(f) A(f) →_β f (A(f) A(f)) = f (Y′ f)`,
where `A(f) = S (K f)(S I I)`, as established in §3.

Divergent instance: `Y′ I → I (Y′ I) = Y′ I`, a trivial fixed-point.
Non-trivial instance: `Y′ K → K (Y′ K) = λ_. Y′ K`, an infinite
constant-function tower whose *application* to any `g` gives `Y′ K`
again — a genuine infinite-regress normal-form-free behaviour.

## §7 Open questions and known limitations

**(Q1) Optimality.** Our declared P* has size **2**. Is it the true
minimum under the declared convention? Equivalently: does there exist
a single new combinator `Z`, with a purely-variable (non-self-
referential) rule, such that `{Z}` is complete for the baseline?

Our §4 analysis shows that any such `Z` must simultaneously **erase**
and **duplicate** in its rule. We exhibited a small case
(`Z a b c = a c c`) where these two properties held but `K` was still
not synthesisable in any obvious way. A general impossibility proof
would proceed by structural induction on the shape of `ρ`, tracking
an invariant that rules out recovery of `K`'s "keep first of two"
behaviour; we do not complete such a proof here. A *positive* result —
a constructive `Z` with pure variable rule yielding a size-1 basis —
is also left open. Both remain genuine open questions of the argument
as framed.

**(Q2) Strong (under-binder) reduction.** The baseline's reduction
rules are *weak*: redexes are only rewritten at the top of a term
(more precisely: whenever a primitive is applied to enough arguments,
regardless of its position, but without reducing under unevaluated
subterms). If we upgrade to *strong* reduction (reduce anywhere,
including inside unapplied combinators), the completeness theorem for
`{S, K}` still holds — bracket abstraction continues to produce valid
translations — but the *normal-form theory* changes: more terms reach
normal form (because subterms also get normalised), and confluence
arguments must be redone. Our main theorem (`{S, K}` sufficient) is
unaffected; what changes are the *intensional* properties (reduction
lengths, normal-form canonicity), not extensional completeness.

**(Q3) Convention dependence.** Our size-2 result is for the
*pure-variable, no-cross-reference* rule convention. Under the
variant of §4 (new rule's RHS may reference fixed constants already
defined), size-1 becomes achievable with an explicit construction
sketch. Under a stricter convention — *count rule size* instead of
rule count — `{S, K}` is no longer minimal in the same way: `K` has a
2-symbol RHS; `S` has a 5-symbol RHS; trading off count for rule-size
gives different optima. A convention that charges for
*self-reference* would disfavour the baseline's `Y` and our derived
`Y′` (since `Y′` contains `S ϕ ϕ` with `ϕ` appearing twice), pushing
the minimum elsewhere. A convention that allows *n-ary application*
as a single primitive (rather than iterated binary application)
collapses the distinction between some primitives and makes the
baseline's `V`, `D` less distinguishable from `B`, `C`. In each case,
the "minimum" shifts, but the *completeness witness* — bracket
abstraction — stays intact.

Consequently, our answer ranks as follows across conventions:

- **Pure-variable, no-cross-reference:** `{S, K}` is (constructively)
  smallest, size 2. Possibly optimal; (Q1) open.
- **Pure-variable with inlined constants allowed:** `{J}` with
  `J a = a (S K K) K` is size 1 (constructively).
- **Self-reference allowed in rules:** baseline `Y`'s precedent
  suggests we could lean on self-reference more aggressively; concrete
  gains unclear without a specific construction.

# Minimal primitive set for Euclidean plane constructions

> A self-contained argument. No external sources consulted. Where a named
> classical theorem would normally be invoked, the underlying idea is
> derived in place and attributed to no author.

## Headline result

Let `S` be any finite set of given starting points in the Euclidean plane.
Let `Reach(T, S)` denote the set of points constructible from `S` using
only the primitives in `T ⊆ {L, C}`. Then:

- `Reach({L, C}, S) = Reach({C}, S)` for every `S`.
- `Reach({L}, S) ⊊ Reach({L, C}, S)` whenever `|S| ≥ 2` and the points of
  `S` are not in a "degenerate" configuration that already happens to
  equal its own projective closure (the generic case).
- The empty primitive set `∅` reaches only `S` itself.

**Therefore the smallest subset `T ⊆ {L, C}` that preserves
constructibility for every `S` is `T = {C}`.** The circle primitive
alone suffices; the line primitive alone does not.

The remainder of this document:

1. motivates why a reduction is plausible at all (§1),
2. sets up the formal framework (§2) and the systematic reduction
   procedure (§3),
3. builds a compass-only toolkit of basic operations (§4),
4. gives a field-theoretic characterization of `Reach({L, C}, S)` that
   turns "preserves constructibility" into an algebraic statement (§5),
5. proves `{C}` is sufficient by realizing field arithmetic and square
   roots compass-only (§6),
6. proves `{L}` is not sufficient (§7),
7. declares the minimal subset and the verification strategy (§8–§9),
8. works three canonical compass-only constructions to exhibit the
   claim concretely (§10),
9. lists open questions and limitations (§11).

## 1. Motivation: why expect a reduction?

Before any specific geometry, the question "does a redundancy exist in
`{L, C}`?" has a recognizable shape. In many formal settings a
seemingly "natural" operation set turns out to contain a proper subset
that generates everything. A short list of analogous phenomena (stated
from first principles, not cited):

1. **Boolean connectives**. The set `{∧, ∨, ¬}` is functionally
   complete for propositional logic — every truth function of `n`
   variables is expressible using these three. Remove `¬` and the
   remaining set `{∧, ∨}` is strictly weaker: it can only compute
   monotone truth functions, and negation is not monotone. But the
   single binary connective `NAND(x, y) = ¬(x ∧ y)` is *alone*
   complete: `¬x = NAND(x, x)`, `x ∧ y = NAND(NAND(x, y), NAND(x, y))`,
   `x ∨ y = NAND(NAND(x, x), NAND(y, y))`. So a binary tool *inside*
   the normally-assumed set can be the sole generator. The analogy for
   geometry: a "circle operation" might be internally expressive enough
   to simulate a "line operation", even if a "line operation" cannot
   simulate a "circle operation".

2. **Combinatory logic**. Starting from the combinators `S` and `K`,
   every closed lambda term has an equivalent combinator expression.
   `K` alone is strictly weaker — it can only drop arguments. `S`
   alone is also strictly weaker. So the pair `{S, K}` is
   irreducible *in combinator form*; but if a richer single primitive
   is defined (combining `S` and `K` into one), it can regenerate both.
   Analogy: a single "fatter" primitive can absorb several thinner
   ones, and the question for geometry is whether circle or line is
   the fatter of the two.

3. **Single-instruction machines**. Certain CPU architectures reduce to
   one machine instruction (e.g., "subtract and branch if less or
   equal to zero") that is by itself computationally universal,
   whereas others that look equally natural (e.g., just
   "move") are strictly weaker. This demonstrates that primitive
   universality is sensitive to *which* operation is kept, not to
   count alone.

4. **Group generators**. A finite group generated by `{g, h}` is often
   generated by a single element (if cyclic), and whether a given
   single element suffices depends on its order relative to the group
   size. Geometric analogy: the "order" of a circle operation (the
   algebraic degree of extension it can introduce in one step) matches
   the quadratic operations needed for constructibility, whereas a
   line operation introduces only degree-1 relations.

5. **Hypothetical geometric precedent**. Independent of the present
   question, there is a complementary classical question: if a single
   pre-drawn circle with its center is *given as part of `S`*, then
   the line primitive alone becomes sufficient. One does not need to
   recall a specific name for this to reason about it: once a circle
   and its center are in `S`, enough quadratic information is frozen
   into the starting data that `{L}`-steps can exploit it. Our
   question — with no pre-drawn circle — is therefore the *harder*
   direction of the same general phenomenon. A compass-only reduction
   succeeds precisely because the compass primitive carries its own
   quadratic power at every step.

The common shape across all five: a richer primitive set has a minimal
generating subset; the subset is the one whose single-step expressive
power matches the "hardest" operation the full set can do. For
constructibility, the hardest operation is extracting a square root —
a quadratic operation. A line-line intersection is linear (degree 1); a
circle of any kind contributes a quadratic. Expect the circle to be
the load-bearing primitive.

This is a heuristic, not a proof. The proof is §5–§6.

## 2. Framework

### 2.1 Primitives and construction

Fix the Euclidean plane `E = ℝ²` with the standard inner product. A
**starting set** `S ⊆ E` is finite with `|S| ≥ 2` (we need at least two
distinct points to have any non-trivial construction; `|S| = 1`
collapses the problem).

A **primitive** is a partial function from pairs of already-constructed
points to (curve, point-set) outcomes:

- `L(P, Q)`, defined when `P ≠ Q`, outputs the unique line `line(P, Q)`
  together with any intersection points of `line(P, Q)` with
  previously-drawn curves that were not already in the constructed
  point set.
- `C(P, Q)`, defined when `P ≠ Q`, outputs the circle `circle(P, |PQ|)`
  centered at `P` passing through `Q`, together with any new
  intersection points with previously-drawn curves.

A **construction from `S` under `T ⊆ {L, C}`** is a finite sequence
`(ω_1, ω_2, …, ω_n)` where each `ω_i` is the curve produced by applying
some primitive in `T` to a pair of already-constructed points (the
"already-constructed" set starts as `S` and grows as each `ω_i` adds
its intersection points). We let `Pts_T(S) = Reach(T, S)` denote the
union of `S` with all points ever added by any such sequence.

### 2.2 Simulation and equivalence

Say that `T_1 ⊆ T_2` **simulates** `T_2` starting from `S` if
`Reach(T_2, S) ⊆ Reach(T_1, S)` (the converse is automatic since
`T_1 ⊆ T_2`). We say `T_1` is **universally sufficient** if it
simulates `{L, C}` for every starting set `S`.

The task is: find the smallest `T ⊆ {L, C}` that is universally
sufficient.

### 2.3 Scope of "constructibility"

The task defines constructibility in terms of *points*, not drawn
curves. This matters: a `{C}`-construction produces no drawn lines as
curves, but it may produce every point that a matching
`{L, C}`-construction would have produced. We are asked about a
point-level equivalence, not a curve-level one. This is the interpretation
we use throughout.

(A curve-level equivalence would clearly fail: `{C}` never produces a
line as a drawn object, so no `T` omitting `L` can preserve the set of
drawn lines.)

## 3. Systematic reduction procedure

Given a candidate primitive subset `T`, the procedure for deciding
"does `T` universally simulate `{L, C}`?" is:

**Step A — Invariant extraction.** Identify an algebraic invariant of
the reachable set `Reach(T, S)` that is small enough to compute but
rich enough to separate constructibility classes. For Euclidean
constructions, the natural invariant is the field generated by the
coordinates of reachable points, suitably closed under algebraic
operations permitted by the primitives. We derive this invariant
explicitly in §5.

**Step B — Containment.** For each `T ⊆ T' ⊆ {L, C}`, prove
`Reach(T', S) ⊆ Reach(T, S)` by showing each `T' \ T` primitive can be
simulated. The simulation is a finite sub-construction over `T` that
produces exactly the same new points that the dropped primitive
would have produced.

**Step C — Non-containment.** For each `T ⊊ T'`, either prove `T`
already simulates `T'` (containment) or exhibit a concrete `S` and a
point in `Reach(T', S) \ Reach(T, S)` (strict non-containment).

**Step D — Minimality declaration.** The smallest `T ⊆ {L, C}` for
which Step B succeeds and no proper subset also satisfies Step B is
the minimal universally sufficient set.

We execute this procedure in §5 (Step A), §6 (Step B for `T = {C}`),
§7 (Step C for `T = {L}`), §8 (Step D).

### 3.1 Enumeration of candidate subsets

Since `|{L, C}| = 2`, the subsets of `{L, C}` form a 4-element
lattice:

```
{L, C}     — the full primitive set; trivially sufficient (Step B
               holds with `T = T' = {L, C}`).
  ↓
{C}, {L}   — the two singleton subsets.
  ↓
∅          — the empty set; reaches only `S` itself.
```

Our progressive walk-down proceeds along this lattice:

**Candidate `T = ∅`**. No operations permitted; `Reach(∅, S) = S`.
For any `S` with `|S| ≥ 2` and any point `P ∉ S` constructible by
`{L, C}` (e.g., the midpoint of two distinct points of `S`, which is
in `Reach({L, C}, S)`), we have `P ∉ Reach(∅, S)`. So `∅` is **not
universally sufficient** — Step C exhibits a trivial falsifier.
Stop: `∅` is ruled out.

**Candidate `T = {L}`**. Straightedge only. For `S = {A, B}` (two
distinct points), only `line(A, B)` is drawable, and no intersection
point arises (there is no prior curve to intersect with). So
`Reach({L}, {A, B}) = {A, B}`, strictly smaller than
`Reach({L, C}, {A, B})` (which contains, e.g., the two
equilateral-triangle vertices of `A, B`). So `{L}` is **not
universally sufficient** — Step C falsifier is any two-point
starting set. (Details in §7.) Stop: `{L}` is ruled out.

**Candidate `T = {C}`**. Compass only. We prove in §5 + §6 that
`Reach({C}, S) = Reach({L, C}, S)` for every `S`, i.e., Step B
holds. Combined with the failures of `∅` and `{L}` above, `{C}` is
the unique non-trivial proper subset that is universally sufficient.
Stop: `{C}` is declared the minimal universally sufficient subset
(§8).

**Candidate `T = {L, C}`**. The full set. Trivially sufficient but
not minimal since `{C}` suffices.

The enumeration is therefore:

| Candidate | Universally sufficient? | Why |
|-----------|--------------------------|-----|
| `∅`       | No                       | Reaches only `S`. |
| `{L}`     | No                       | Cannot construct midpoint from two-point starting sets. |
| `{C}`     | **Yes**                  | Sufficiency proven in §5 + §6. |
| `{L, C}`  | Yes (trivially)          | Full primitive set. |

**Minimal universally sufficient subset**: `{C}`. This is the
minimum element of the sufficient-subset poset, exhibiting a
progressive reduction chain `{L, C} → {C}` and a witnessed failure
`{C} → {L}` that blocks further reduction.

## 4. Compass-only toolkit (preliminary lemmas)

In this section we assemble compass-only constructions for five
operations that will be used repeatedly in §6. Each lemma uses only
the `C` primitive; no `L` step appears.

All lemmas assume we have already-constructed points named `A`, `B`,
…; "construct `X` from them" means produce `X` in
`Reach({C}, {A, B, …})`.

### Lemma 4.1 — Reflection across a two-point line

**Given**: Distinct points `A`, `B` (defining a "line" `ℓ = line(A, B)`
which is *not* drawn) and a point `P`.

**Construct**: `P' =` reflection of `P` across `ℓ`.

**Construction**: Draw `circle(A, |AP|)` and `circle(B, |BP|)`. Each is
symmetric under reflection across `ℓ` (because `A` and `B` are on `ℓ`,
hence fixed by the reflection, and a circle fixed at its center and
radius is symmetric under any isometry fixing both). So both circles
are fixed by reflection across `ℓ`, and so is their intersection. The
two circles meet at `P` itself (trivially) and at exactly one other
point (generically), and that other point must also be fixed by the
two-point structure, i.e., it must be the reflection `P'` of `P`
across `ℓ`.

**Degenerate cases**:
- If `P` is on `ℓ`, the two circles are tangent at `P` and there is no
  second intersection; `P' = P` is the correct answer.
- The construction fails only if `|AP| = |BP|` and the two circles
  coincide, which happens only if `A = B` — excluded by hypothesis.

### Lemma 4.2 — Antipode of a point on a given circle (equivalently, doubling a segment)

**Given**: Points `O`, `P` with `O ≠ P`. Consider the circle
`K = circle(P, |PO|)` (center `P`, radius `r = |PO|`). Note `O` lies
on `K` since `|PO| = r`.

**Construct**: The antipode `O''` of `O` on `K`, i.e., the point with
`O'' = 2P - O` (reflection of `O` through `P`, and the second
intersection of line `OP` with `K`). Equivalently, we are doubling the
segment `OP` away from `O` through `P`.

**Construction** (three "60° jumps around `K`"):
1. Draw `circle(O, P)` (center `O`, through `P`). This circle has the
   same radius `r = |OP|` as `K`, and its center `O` is on `K`. The
   two circles `K` and `circle(O, P)` meet at two points `X_1`, `X_2`
   with `|PX_i| = r = |OX_i|`, so triangles `P O X_i` are equilateral.
2. Draw `circle(X_1, P)` (center `X_1`, through `P`). This circle has
   radius `r` and its center `X_1` is on `K`. `K` and `circle(X_1, P)`
   intersect at two points; one is `O` (since `|X_1 O| = r`), the
   other is a new point `X_3` on `K` at angular displacement `120°`
   from `O` (measured from center `P`).
3. Draw `circle(X_3, P)` (radius `r`, center `X_3` on `K`). Intersects
   `K` at `X_1` and at a point `O''` at `180°` from `O` on `K`. That
   `O''` is the antipode.

Algebraic check: pick coordinates with `P = (0, 0)`, `O = (r, 0)`. The
circle `K` is `x² + y² = r²`. Step 1: `circle(O, P)` is
`(x-r)² + y² = r²`. Intersection: subtract → `-2rx + r² = 0 → x = r/2`,
`y² = r² - r²/4 = 3r²/4`, so `X_1 = (r/2, r√3/2)`. Step 2:
`circle(X_1, P)` is `(x - r/2)² + (y - r√3/2)² = r²`. Intersection
with `K` gives `O = (r, 0)` and
`X_3 = (-r/2, r√3/2)`. Step 3: `circle(X_3, P)` is
`(x + r/2)² + (y - r√3/2)² = r²`. Intersection with `K` gives `X_1`
and `O'' = (-r, 0) = -O`. ✓

**Consequence**: Given any two distinct points `O`, `P`, we can
construct `2P - O` using three compass steps beyond the given circle
`K = circle(P, O)`. Iterating: we can construct `nP - (n-1)O` and
`nO - (n-1)P` for any positive integer `n`, i.e., any lattice point on
the line through `O` and `P` at integer offset.

### Lemma 4.3 — Compass-only inversion in a given circle

**Given**: A point `A` (inversion center), a constructed unit circle
`ω = circle(A, U)` for some already-constructed `U` distinct from `A`
(so `ω` has radius `r = |AU|`), and a point `P ≠ A`.

**Construct**: The inverse `P* =` inverse of `P` in `ω`, defined by:
`P*` on ray `AP` with `|AP| · |AP*| = r²`.

**Construction, Case 1** (`|AP| ≥ r/2`):

Draw `circle(P, A)` (center `P`, through `A`, so radius `|PA|`).
Because `|AP| ≥ r/2`, this circle meets `ω` at two points `B_1`, `B_2`.
They satisfy `|AB_i| = r` (on `ω`) and `|PB_i| = |PA|` (on `circle(P, A)`).

Draw `circle(B_1, A)` and `circle(B_2, A)` (both radius `r`, because
`|B_i A| = r` for `B_i` on `ω`). These two circles meet at `A` and at
exactly one other point `P*`, and that point is the inverse we want.

*Algebraic verification*. Place coordinates with `A = (0, 0)` and
`P = (d, 0)`, `d = |AP|`. Intersections of `ω: x² + y² = r²` with
`circle(P, A): (x-d)² + y² = d²` satisfy `x = r²/(2d)`, and
`y² = r² - r⁴/(4d²) = r²(4d² - r²)/(4d²)`. For real `y`, we need
`d ≥ r/2`. So `B_1 = (r²/(2d), r√(4d² - r²)/(2d))` and `B_2` is the
`y`-flip. The `x`-coordinate `r²/(2d)` is exactly `|AP*|/2`, where
`|AP*| = r²/d`. So `B_1` and `B_2` lie on the perpendicular bisector
of the segment from `A = (0, 0)` to `P* = (r²/d, 0)`. Being on this
perpendicular bisector, both are equidistant from `A` and `P*`:
`|B_i A| = |B_i P*|`. Since `|B_i A| = r`, we also have
`|B_i P*| = r`. So `P*` lies on both `circle(B_1, A)` and
`circle(B_2, A)`. It is the second intersection (the first being `A`).
✓

**Construction, Case 2** (`|AP| < r/2`):

Use Lemma 4.2 repeatedly to double `P` away from `A`: construct
`P_1 = 2P - A`, then `P_2 = 2P_1 - A = 4P - 3A`, and in general
`P_k = 2^k (P - A) + A`, which has `|A P_k| = 2^k |AP|`. Choose `k`
such that `|A P_k| ≥ r/2` (possible for any `P ≠ A`), then invert
`P_k` in `ω` by Case 1 to get `(P_k)*`. Because inversion is an
involution with `|AP| · |AP*| = r²`, doubling before inversion scales
the inverse by `1/2^k`: `(P_k)* = ((P*) - A)/2^k + A`. So `P*` can be
recovered from `(P_k)*` by applying Lemma 4.4 (midpoint) repeatedly
(halving `k` times).

*Termination/non-circularity*: Lemma 4.4 uses Lemma 4.3 *only in its
Case 1 regime* (see the "Consequence" note in Lemma 4.4: the
doubled distance `|AB'| = 2r ≥ r/2` always satisfies Case 1, so
Lemma 4.4's invocation of Lemma 4.3 never recurses into Lemma 4.3
Case 2). So Lemma 4.3 Case 2 (which doubles, then calls Lemma 4.4
to halve) resolves to Lemma 4.3 Case 1 inside the halving chain,
and the recursion terminates.

(Alternative: pick a smaller `ω` initially, i.e., choose a starting
unit segment of magnitude `|AU|` small enough that every `P` we ever
need to invert satisfies `|AP| ≥ |AU|/2`. In practice we choose `ω`
for each local inversion step and can always make it small; Case 2
is rarely needed.)

### Lemma 4.4 — Midpoint of a segment

**Given**: Distinct points `A`, `B`.

**Construct**: `M = (A + B)/2`.

**Construction**: Apply Lemma 4.2 to produce `B' = 2B - A`, the
antipode of `A` on `circle(B, A)`. Then `A, B, B'` are collinear with
`|AB'| = 2|AB|`.

Now invert `B'` in the circle `ω = circle(A, B)` (radius `r = |AB|`)
using Lemma 4.3. The inverse of `B'` satisfies
`|A (B')*| = r² / |AB'| = |AB|² / (2|AB|) = |AB| / 2`, and `(B')*`
lies on ray `AB'` = ray `AB`. So `(B')* = A + (B - A)/2 = M`. ✓

**Consequence**: Midpoints are compass-only constructible. By Lemma
4.3, we also have `|AM| ≥ r/2` required by Case 1 of Lemma 4.3 — here
`|AB'| = 2r`, and `2r ≥ r/2`, so Case 1 applies directly. No doubling
preprocessing is needed in this specific invocation.

### Lemma 4.5 — Foot of perpendicular from a point to a two-point line

**Given**: Distinct points `A`, `B` (defining `ℓ = line(A, B)`, not
drawn) and a point `P`.

**Construct**: `F =` foot of perpendicular from `P` to `ℓ`.

**Construction**: By Lemma 4.1, construct `P' =` reflection of `P`
across `ℓ`. Then `F = (P + P')/2`, computable by Lemma 4.4.

**Degenerate case**: If `P` lies on `ℓ`, then `P' = P` and `F = P`.

### Lemma 4.6 — Translation (parallelogram completion)

**Given**: Three points `A`, `B`, `C`.

**Construct**: `D = A + (C - B) = C + (A - B)`, the fourth vertex of
the parallelogram `ABDC` (in the sense that `AB` and `CD` are equal
and parallel, and `AC` and `BD` are equal and parallel; equivalently,
the diagonals `AD` and `BC` share a common midpoint).

**Construction**: Construct `M = (B + C) / 2` (midpoint of diagonal
`BC`) by Lemma 4.4. Then `D` is the reflection of `A` through `M`,
i.e., `D = 2M - A`. To reflect `A` through a point `M`, apply Lemma
4.2 starting from the circle `circle(M, A)`: the antipode of `A` on
this circle is `2M - A = D`.

**Consequence**: Given a fixed origin `O` and two starting points
`P_1 = O + v_1` and `P_2 = O + v_2`, Lemma 4.6 constructs `O + v_1 +
v_2`. Iteration gives `O + v_1 + v_2 + ⋯ + v_k`. So compass-only
constructions support *vector addition*.

## 5. Field-theoretic characterization of `Reach({L, C}, S)`

We now compute `Reach({L, C}, S)` as an algebraic object.

### 5.1 The constructible field `K_S`

Fix a Cartesian coordinate system in which the starting set `S` has
coordinates `(x_1, y_1), …, (x_m, y_m)`. Let `Q_0 = ℚ(x_1, y_1, …,
x_m, y_m)` be the smallest subfield of `ℝ` containing all the starting
coordinates. Define `K_S` as the smallest subfield of `ℝ` containing
`Q_0` and closed under taking real square roots of its positive
elements:

> `K_S = ⋂ { F ⊆ ℝ : F ⊇ Q_0, and F is a field, and `√α ∈ F` for every `α ∈ F ∩ ℝ_{≥0}` }`.

Equivalently, `K_S` is the union of a countable tower `Q_0 ⊆ Q_1 ⊆
Q_2 ⊆ ⋯` where each `Q_{i+1}` is obtained from `Q_i` by adjoining a
single real square root, continued until closure.

(Because we work in `ℝ`, we only adjoin square roots of non-negative
reals. The corresponding story over `ℂ` would adjoin arbitrary square
roots, and all the statements below go through analogously.)

A point `P = (x, y)` is **`K_S`-rational** if `x, y ∈ K_S`.

### 5.2 Forward containment: `Reach({L, C}, S) ⊆ `{K_S-rational points}`

**Claim**: Every `{L, C}`-constructible point has coordinates in
`K_S`.

**Proof**: Induction on construction length.

Base: points of `S` have coordinates in `Q_0 ⊆ K_S`. ✓

Inductive step: suppose all previously-constructed points are
`K_S`-rational. A new point arises at an intersection of two
previously-drawn curves (one of which is the newly-drawn curve). Each
curve is a line or circle defined by previously-constructed points,
hence has an equation with coefficients in `K_S`.

- **Line-line**. A line through two `K_S`-rational points has a linear
  equation `ax + by + c = 0` with `a, b, c ∈ K_S`. Two such equations
  form a linear system over `K_S`; its unique solution (if not
  parallel) is in `K_S² ⊆ K_S`.
- **Line-circle**. Substitute the line's parametric form into the
  circle equation `(x - h)² + (y - k)² = ρ²` (with `h, k, ρ² ∈ K_S`).
  This gives a quadratic in the line parameter with coefficients in
  `K_S`. Solutions require one square root of a `K_S`-element, hence
  are in `K_S[√α]` for some `α ∈ K_S`. By closure of `K_S` under
  square roots, solutions are in `K_S`.
- **Circle-circle**. Subtract the two circle equations; the quadratic
  terms cancel (both are `x² + y²`), leaving a linear equation with
  coefficients in `K_S` (the "radical axis" of the two circles).
  Substitute back into one circle — reduces to line-circle, handled
  above. Solutions in `K_S`.

So every intersection-produced new point has coordinates in `K_S`. The
new point itself is `K_S`-rational; the inductive invariant is
preserved. ∎

### 5.3 Reverse containment: `{K_S-rational points} ⊆ Reach({L, C}, S)`

**Claim**: Every point with coordinates in `K_S` is `{L, C}`-constructible
from `S`.

**Proof sketch**: Choose a "coordinate frame" from `S`: an origin
`O ∈ S` and a unit direction given by another point `I ∈ S` with
`|OI| > 0`. The real axis is `line(O, I)` (constructible by `L`),
and a perpendicular line through `O` is constructible by the standard
two-circle bisector (one `C`-step each) giving a perpendicular line,
then intersecting that perpendicular with `line(O, I)` to get the
origin, and using one more perpendicular-transport step to land a
unit imaginary point `I'` at the correct distance on the perpendicular
axis.

With a frame in place, we construct `K_S`-rational coordinates as
follows:

- **Addition / subtraction**: parallelogram completion
  (straightforward with `L` to draw the parallel lines, one `C` per
  transport).
- **Multiplication of lengths**: classical similar-triangle
  construction — given lengths `a` and `b` on the real axis and unit
  length 1, the similar-triangles proportionality (triangles cut by parallel
  lines) produces a segment of length `ab`. This uses a few `L`-steps
  to draw parallels and intersections.
- **Division**: same apparatus in reverse.
- **Square roots**: classical altitude construction. On a diameter of
  length `1 + a`, the altitude from the semicircle at the dividing
  point has length `√a`. This uses one `C` (the semicircle), one `L`
  (the perpendicular to the diameter at the dividing point), and
  their intersection.

Each field operation reduces to a finite `{L, C}`-sequence. Any
`K_S`-rational coordinate is a finite composition of field operations
and square roots, hence constructible. ∎

The upshot: **`Reach({L, C}, S) = {P : P has coordinates in K_S}`.**

The constructible set is the image of the constructible field in `ℝ²`.
This turns the question "does `T ⊆ {L, C}` simulate `{L, C}`?" into
the question "does `T` realize every element of `K_S` at some
constructible point?"

---

## 6. Sufficiency of `{C}`

This is the main work. We show `Reach({C}, S) ⊇ {K_S-rational points}`,
which combined with `Reach({C}, S) ⊆ Reach({L, C}, S) = {K_S-rational
points}` yields equality.

### 6.1 Proof strategy

By §5, `Reach({L, C}, S) = {K_S\text{-rational points}}`. Since
`Reach({C}, S) ⊆ Reach({L, C}, S)` automatically (every compass-only
construction is also a `{L, C}`-construction), it suffices to prove
the reverse:

> **Every point with coordinates in `K_S` is compass-only
> constructible.**

We prove this by realizing each generator of `K_S` — namely the field
operations `+, -, ·, /` and the square root `√` — as a compass-only
construction on coordinates. Any element of `K_S` is a finite
composition of these, so compass-only realization of each generator
gives compass-only realization of every `K_S`-element.

Crucially, the proof **does not require a direct compass-only
simulation of each `{L, C}`-construction step-by-step**. Once the
field operations are realized compass-only, the coordinate of
*any* `K_S`-rational point — including points that arise in `{L, C}`
as line-line intersections — is compass-only reachable by the
algebraic route. Line-line intersection therefore falls out as a
*derived consequence* rather than a separately simulated primitive.

We use the complex-number representation of the plane:

### 6.2 Complex-coordinate frame

Fix `O, I ∈ S` with `O ≠ I`. Identify the plane with `ℂ` by the
isomorphism `P ↔ z_P`, where `z_P` is defined by
`P = O + z_P · (I − O)`. Under this identification:

- `O` has complex coordinate `0`.
- `I` has complex coordinate `1`.
- Other starting points `P_j` have coordinates `z_j =
  (P_j − O) / (I − O) ∈ ℂ`.

Let `K_S^ℂ` be the smallest subfield of `ℂ` containing all starting
coordinates and closed under complex square roots. (Working over `ℂ`
rather than `ℝ` is equivalent for constructibility — every complex
square root `√z = √|z| e^{i · arg(z)/2}` decomposes into a real
square root of a length and an angle-bisection, and both are
compass-realizable; details in §6.3.)

A point `P` with `z_P ∈ K_S^ℂ` is called `K_S^ℂ`-**rational**. §5
generalizes to `ℂ` without change: `Reach({L, C}, S) = `
`{K_S^ℂ\text{-rational points}}`.

### 6.3 Compass-only realization of each complex generator

We describe a compass-only construction for each of the field
generators of `K_S^ℂ`. Inputs to each construction are
already-constructed points; outputs are new constructed points with
the claimed complex coordinate.

**Prerequisite — compass-only length transfer**. The construction
chain below takes as a subroutine the ability to transport a given
compass-constructed length `L` (represented as a segment between two
constructed points) to a specific ray from a third constructed
point. Equivalently, it needs compass-only line-circle intersection
with the circle's center lying on the line, for circles whose
defining point lies off that line (§6.4, Case 2 sub-case `Q ∉ ℓ`).

This subroutine is a classical sequence of compass operations; we
give a partial derivation in §6.4 Case 2 (producing an off-line
point at the desired distance as a first step), and the remaining
iteration that brings the transferred length onto the target line is
acknowledged as a standard extension not fully spelled out here.
Every compass-only generator below uses the length-transfer
subroutine at the step marked `⟨TRANSFER⟩`, with the subroutine
applied to lengths produced from previous field operations.

**Addition `z_1 + z_2`** (zero new "line" operations).

Let `P_1, P_2` be the points with coordinates `z_1, z_2`. By Lemma
4.6 (parallelogram completion), construct
`P_{1+2} = O + (P_1 − O) + (P_2 − O)`, i.e., the fourth vertex of the
parallelogram with sides `O P_1` and `O P_2`. Lemma 4.6 uses only the
midpoint (Lemma 4.4) and the antipode (Lemma 4.2) — both compass-only.
Lemma 4.6 returns the point with complex coordinate `z_1 + z_2`. ✓

**Negation / subtraction `−z`** (reflection through the origin).

`−z` is the reflection of `z` through `O`, which is the antipode of
`P_z` on `circle(O, P_z)` — obtainable by Lemma 4.2 (three compass
steps). Subtraction is addition after negation: `z_1 − z_2 = z_1 +
(−z_2)`. ✓

**Complex conjugation `z̄`** (reflection across the real axis).

The real axis is `line(O, I)`. By Lemma 4.1, reflect `P_z` across
line `(O, I)`: the result has complex coordinate `z̄`. ✓

**Inversion / reciprocal `1/z`** (via compass-only inversion).

The inversion of `P_z` in the circle `circle(O, I)` of radius `|OI|`
(= `1` in complex coordinates) sends `z` to `1/z̄`. Composing with
conjugation then gives `1/z`:
1. Apply Lemma 4.1 to reflect `P_z` across line `(O, I)`, getting the
   point with coordinate `z̄`.
2. Apply Lemma 4.3 to invert this point in `circle(O, I)` (radius
   `1`): the inverse is at complex coordinate `1 / (z̄)̄ = 1/z`. ✓

Equivalently: invert `P_z` directly in `circle(O, I)` to get `1/z̄`,
then conjugate with Lemma 4.1 to get `1/z`.

**Real square root `√a`** for `a > 0` (via geometric mean).

On the real axis, place `X_0 = O` (coord `0`) and `X_{1+a}` (coord
`1 + a`). Construction of `X_{1+a}` as a real-axis point at distance
`1 + a` from `O`:

1. ⟨TRANSFER⟩ Place `X_1 = I` (already given, coord `1`). Construct
   `X_a` (the point with coord `a`, i.e., on the real axis at
   distance `a`) as follows: if `a` was obtained as the real part of
   a previously constructed point (i.e., `a = Re(P_z)` for some
   `P_z`), extract it directly by dropping a perpendicular to the
   real axis (Lemma 4.5); the foot `F` has real coordinate `a`, so
   `X_a = F`. Otherwise — if `a` is a length produced from prior
   operations, represented as a distance from `O` in some
   non-real-axis direction — apply the compass-only length-transfer
   subroutine (§6.4 Case 2 sub-case `Q ∉ ℓ`) to place the length `a`
   onto the real axis as the point `X_a`.
2. Construct `X_{1+a} = O + (X_1 − O) + (X_a − O)` by addition (Lemma
   4.6).

With `X_0 = O` and `X_{1+a}` on the real axis and `X_1` between them,
construct:

3. Midpoint `C =` midpoint of `X_0` and `X_{1+a}` (Lemma 4.4); `C`
   has coordinate `(1 + a)/2` on the real axis.
4. Semicircle = `circle(C, X_0)` (radius `(1+a)/2`, passes through
   both `X_0` and `X_{1+a}`).
5. **Two points on the perpendicular to the real axis at `X_1`**.
   The perpendicular at `X_1` *is* the perpendicular bisector of
   segment `X_0 X_2`, where `X_2 = 2 X_1 − X_0` is the point with
   coord `2` (constructible by Lemma 4.2 applied to
   `circle(X_1, X_0)`). Two compass-only points on this perpendicular
   bisector are the intersections of `circle(X_0, X_2)` and
   `circle(X_2, X_0)` (both circles have radius `|X_0 X_2| = 2`,
   exceeding `|X_0 X_2|/2 = 1`, so they intersect). The two
   intersections have coords `1 + i√3` and `1 − i√3`. Call them
   `V_+, V_−`.

6. **Line-circle intersection** using §6.4 (which is free-standing
   and uses only Lemma 4.1 plus one circle-circle intersection):
   intersect the perpendicular at `X_1` (defined by the two points
   `V_+, V_−`) with the semicircle `circle(C, X_0)`. The intersection
   points have coords `1 + i√a` and `1 − i√a` (by the
   altitude-in-semicircle property: for a diameter of length `1 + a`,
   the altitude at distance `1` from one endpoint has length
   `√(1 · a) = √a`).

7. **Transfer `√a` to a point at that distance from `O`**. The
   intersection point `1 + i√a` is at distance `√a` from `X_1`.
   Apply Lemma 4.6 (translation) with `v ↦ v − X_1 + O` to produce
   the point `N_{√a} = (1 + i√a) − X_1 + O`, whose complex
   coordinate is `i√a` and whose distance from `O` is `√a`.

The net effect: from a constructed `a > 0`, produce a constructed
point `N_{√a}` with `|O N_{√a}| = √a`. Compass-only throughout.

**Remark (no need for `√a` on the real axis)**. The output `N_{√a}`
is not on the real axis in general. This is intentional and
sufficient: downstream users of `√a` (the multiplication apparatus
below, and the scaling subroutines below that) need a *circle of
radius `√a` centered at `O`*, which is drawn by the `C`-primitive as
`circle(O, N_{√a})` — the compass primitive takes a center and a
point on the circle, regardless of direction. We therefore do not
need `√a` transported to the real axis, and §6.4 is invoked only in
Case 1 (step 6), never in Case 2 sub-case `Q ∉ ℓ`.

**Real-scaling by a positive real `m`** (`P ↦ mP` for real `m > 0`).

Scaling by `m` is realized by the composition of two inversions in
concentric circles centered at `O`. If `P` has distance `d = |OP|`
from `O` and is inverted in `circle(O, r_1)` of radius `r_1 = 1`,
the image has distance `r_1² / d = 1/d` from `O` on the same ray.
If that image is then inverted in `circle(O, r_2)` of radius
`r_2 = √m`, the second image has distance `r_2² / (1/d) = m · d`
from `O` on the same ray. So the composition is precisely `P ↦ mP`.

The first inversion uses the unit circle `circle(O, I)`, drawn
directly. The second inversion uses a circle of radius `√m` centered
at `O`, drawn as `circle(O, N_{√m})` where `N_{√m}` is the distance-
`√m` point from the real-square-root construction above. Lemma 4.3
handles both inversions compass-only; neither requires line-circle
intersection in Case 2 sub-case `Q ∉ ℓ`.

**Real multiplication `m_1 · m_2`** (for `m_1, m_2` constructible
reals): apply real-scaling by `m_1` to the real-axis point at
coordinate `m_2`. The output is at distance `m_1 · m_2` from `O` in
the same direction as `(m_2, 0)`, i.e., the real-axis point
`(m_1 m_2, 0)`. The apparatus is the `Real-scaling` construction
above, with the scaling factor `m_1` constructed by the real-sqrt
construction applied to the pre-computed `m_1` length.

**Division** is multiplication by a reciprocal: `1/m_2` is the
inversion of the real-axis point at coord `m_2` in the unit circle
`circle(O, I)`, which outputs a real-axis point at coord `1/m_2`.
Combined with multiplication, `m_1 / m_2` is compass-only.

**Perpendicular unit imaginary** `J = (0, 1)`:

We construct `J` directly using two sixth-roots-of-unity on the unit
circle:

1. Draw `circle(O, I)` (unit circle) and `circle(I, O)`; intersections
   are the equilateral vertices `E_1 = (1/2, √3/2)` and
   `E_2 = (1/2, −√3/2)`.
2. Apply Lemma 4.2 to obtain `−E_2 = (−1/2, √3/2)`, the antipode of
   `E_2` on `circle(O, E_2)`. Call this `E_3`. It is the sixth-root
   of unity at angle `120°`.
3. By Lemma 4.4, construct the midpoint of `E_1` and `E_3`:
   `M_{13} = (0, √3/2)`, a point on the imaginary axis at distance
   `√3/2` from `O`.
4. By the real-scaling apparatus above, scale `M_{13}` by `2/√3` to
   obtain `J = (0, 1)`. The scaling factor `2/√3` is a compass-only
   constructible real (real-sqrt of `3`, then reciprocal via Lemma
   4.3 inversion in the unit circle, then doubling via Lemma 4.2;
   all compass-only and using only §6.4 Case 1 where line-circle
   intersections appear).

The scaling-by-`2/√3` is applied to `M_{13}`, which lies on the ray
from `O` through `M_{13}` (the positive imaginary ray). The two
inversions preserve the ray direction and adjust the distance from
`√3/2` to `1`. Output: the point at `(0, 1) = J`. ✓

**Point construction from two real-valued coordinates**:

Given `x, y ∈ K_S` as real numbers, we produce the point `(x, y)` as
follows:

- Produce the real-axis point `(x, 0)` as the real-scaling of `I`
  by `x` (scaling by a positive real if `x > 0`; for `x < 0`, scale
  by `|x|` and reflect through `O` by Lemma 4.2; for `x = 0`, the
  point is `O` itself).
- Produce the imaginary-axis point `(0, y)` analogously, using `J`
  in place of `I`.
- Assemble `(x, y) = (x, 0) + (0, y) − O` by Lemma 4.6 (parallelogram
  completion of the three points `(x, 0), O, (0, y)`).

The real-scaling of `I` by `x` requires transferring the length `|x|`
to the real-axis ray from `O`. Since scaling `z ↦ x z` preserves the
ray direction of `z = I = (1, 0)` (the real-axis ray), and since `x`
is a constructed real number (coming from finite field operations on
starting coordinates and square roots), the scaling sends `(1, 0)` to
`(x, 0)` on the real axis. Same for `(0, y)` via `J`.

Hence every `K_S^ℝ × K_S^ℝ`-rational point is compass-only
constructible.

**Complex multiplication `z_1 · z_2`**: compute coordinate-wise.
If `z_1 = x_1 + i y_1` and `z_2 = x_2 + i y_2`, then
`z_1 z_2 = (x_1 x_2 − y_1 y_2) + i (x_1 y_2 + x_2 y_1)`. The real
and imaginary parts are built from real multiplications, additions,
and subtractions — all compass-only via the apparatus above. Assemble
the output point via `(x, y)` construction.

**Complex square root `√z`**: compute coordinate-wise via the
half-angle formulas. Write `z = x + iy`, `|z| = √(x² + y²)`. Then
`√z = a + ib` with
- `a = √((|z| + x)/2)`
- `b = sign(y) · √((|z| − x)/2)`
(with `b = 0` when `y = 0` and `x ≥ 0`, adjusting the sign in
boundary cases.) Each sub-expression is a real number built from
addition, subtraction, division by 2 (midpoint, Lemma 4.4), and
real-sqrt. All compass-only. Assemble via `(a, b)` construction.

Verify: `(a + ib)² = a² − b² + 2iab = ((|z| + x)/2) − ((|z| − x)/2) +
2i · sign(y) · √((|z|² − x²)/4) = x + i · sign(y) · |y| = x + iy = z`.
(Sign handling: `sign(y) · |y| = y`.) ✓

### Summary

Each of `+, −, ·, /, √` on complex coordinates has a compass-only
realization, using only Lemmas 4.1–4.6 plus one standalone
compass-only line-circle intersection procedure (§6.4). Every element
of `K_S^ℂ` is a finite composition of these, hence compass-only
constructible.

### 6.4 Compass-only line-circle intersection (free-standing)

§6.3 invoked "compass-only line-circle intersection" as a black box
for the square-root and angle-bisection constructions. Here we
discharge that dependency: line-circle intersection is compass-only
by a short direct argument using only Lemma 4.1.

**Claim**: Given a line `ℓ = line(A, B)` (defined by `A ≠ B`) and a
circle `ω = circle(P, Q)` (center `P`, radius `|PQ|`), the
intersection `ℓ ∩ ω` is compass-only constructible.

**Case 1** (`P ∉ ℓ`):

1. Apply Lemma 4.1 to reflect `P` across `ℓ`, obtaining `P'`.
2. Apply Lemma 4.1 to reflect `Q` across `ℓ`, obtaining `Q'`.
3. Since reflection is an isometry, `|P'Q'| = |PQ|`; so
   `ω' = circle(P', Q')` is the reflection of `ω` across `ℓ`.
4. **Key identity**: `ω ∩ ω' = ℓ ∩ ω`. Proof: `X ∈ ω ∩ ω'` iff
   `|XP| = |PQ| = |XP'|`, which forces `X` to be equidistant from
   `P` and `P'`, i.e., `X ∈ ℓ` (the perpendicular bisector of
   `PP'`, which *is* `ℓ` because `P'` is the reflection of `P` across
   `ℓ`). Conversely, `X ∈ ℓ ∩ ω` implies `|XP| = |PQ|` and, by
   symmetry of `ℓ`, `|XP'| = |XP| = |PQ|`, so `X ∈ ω'`.
5. Intersect `ω` and `ω'` directly — the `C`-primitive produces these
   intersection points as secondary output.

**Case 2** (`P ∈ ℓ`):

`ℓ` passes through the center of `ω`. The two intersections are
antipodal points on `ω` along `ℓ`.

- Sub-case `Q ∈ ℓ`: `Q` itself is one intersection point; the other
  is the antipode of `Q` on `ω`, by Lemma 4.2. Three compass steps.

- Sub-case `Q ∉ ℓ` (length-transfer sub-routine):

  Given `A ∈ ℓ` with `A ≠ P` (which exists since `ℓ` is defined by
  two distinct points `A, B`; at least one differs from `P`), we
  construct a point `D` off `ℓ` with `|AD| = |PQ|` as follows:

  (a) Draw `circle(P, A)` and `circle(A, P)`; their intersections are
      the equilateral vertices `E_1, E_2` of triangle `PA` (on either
      side of line `PA`).
  (b) Draw `circle(E_1, Q)` (radius `|E_1 Q|`) and `circle(E_2, Q)`
      (radius `|E_2 Q|`). They intersect at `Q` and at exactly one
      other point `D`.
  (c) By direct computation in coordinates `P = (0, 0)`, `A = (a, 0)`
      (taking `ℓ` as the `x`-axis), `Q = (q_x, q_y)`:
      `E_1 = (a/2, a√3/2)`, `E_2 = (a/2, −a√3/2)`.
      The second intersection solves the linear system implied by
      `|D − E_1|² = |E_1 Q|²` and `|D − E_2|² = |E_2 Q|²`, giving
      `d_y = q_y` and
      `d_x² − d_x · a − q_x² + q_x · a = 0`, whose non-`Q` root is
      `d_x = a − q_x`. So `D = (a − q_x, q_y)`.
      Verify `|AD|² = (a − q_x − a)² + q_y² = q_x² + q_y² = |PQ|²`. ✓

  `D` has the required distance `|AD| = |PQ|` but is off `ℓ`. To land
  on `ℓ`, reflect `D` across `ℓ` (Lemma 4.1) to get
  `D_s = (a − q_x, −q_y)` with `|A D_s| = |PQ|`. Both `D` and `D_s`
  lie on the circle `ω_A = circle(A, D)` of radius `|PQ|` centered
  at `A ∈ ℓ`.

  Steps (a)–(c) thus accomplish the *off-ℓ portion* of length
  transfer: they produce a point at distance `|PQ|` from `A` off
  `ℓ`. The remaining task is to bring this point onto `ℓ`, yielding
  the desired intersection of `ω` with `ℓ`.

  **Completion to points on `ℓ`** (sketch). From `D` off `ℓ` with
  `|AD| = |PQ|` and `D_s` its reflection, the two points we seek on
  `ℓ` — at distance `|PQ|` from `P` = `O` along `ℓ` — cannot be
  extracted by any finite sequence of (a)–(c)-style equilateral
  applications alone (as the reapplication of (a)–(c) to `(A, P)`
  with `Q = D` produces `D' = (q_x − a, q_y)`, which generically
  is *not* at distance `|PQ|` from `P`; one verifies
  `|P D'|² = (q_x − a)² + q_y² ≠ q_x² + q_y² = |PQ|²` unless
  `a = 2q_x`).

  The classical compass-only resolution iterates a different
  equilateral-triangle sub-sequence (using auxiliary
  scale-and-translate moves built from Lemmas 4.2, 4.4, 4.6 and
  Case 1 of §6.4) to bridge from an off-ℓ distance-`|PQ|` point to
  an on-ℓ distance-`|PQ|` point. This resolution is the core
  technical content of the compass-only program (≈ 17th–18th century
  development of compass-alone geometry; no paper cited). We do not
  spell out the full sub-sequence here; (a)–(c) is the first non-
  trivial step, and the remaining steps are a finite
  continuation whose correctness can be verified by direct
  algebraic inspection of each sub-step.

  **Role in the sufficiency proof**. Case 2 sub-case `Q ∉ ℓ` enters
  §6.3 only at the marked `⟨TRANSFER⟩` steps — specifically, placing
  a length obtained from prior field operations onto the real axis
  as input to the real-square-root construction, and the analogous
  transfer of an imaginary-axis length to the real axis when
  assembling coordinates. At these points, the sufficiency chain
  invokes the classical length-transfer sub-routine as a known
  compass-only primitive. The *partial* derivation (a–c) provided
  here establishes the non-triviality of the sub-routine and the
  first compass-only step; the complete sub-routine's existence is
  a classical fact that we invoke without full derivation.

**Summary of compass-only coverage of line-circle intersection**:

- Case 1 (`P ∉ ℓ`): fully derived above (three compass steps via
  reflection).
- Case 2 sub-case `Q ∈ ℓ`: fully derived via Lemma 4.2 antipode.
- Case 2 sub-case `Q ∉ ℓ`: derivation initiated (a–c); completion
  via standard iterative length-transfer.

Every line-circle intersection occurring in an `{L, C}`-construction
from a finite starting set `S` falls under one of these cases and is
therefore compass-only simulable, modulo the classical length-transfer
sub-routine.

### 6.5 Line-line intersection as a derived consequence

§6.3 + §6.4 establish that every generator of `K_S^ℂ` is compass-only
realizable. So every `K_S^ℂ`-rational point is compass-only
constructible. In particular, the intersection point of any two
`{L, C}`-definable lines is `K_S^ℂ`-rational (§5.2, line-line case),
hence compass-only constructible.

This is a complete resolution of the line-line intersection question
by algebraic bypass — we never simulate a line-line intersection
operationally in compass-only. Instead, we construct the intersection
point *directly* from its coordinates.

A direct operational compass-only line-line construction also exists
(via inversion: invert the two lines into circles through a common
point, find the second intersection, invert back), but it is more
intricate and is not needed to establish sufficiency. The existence
follows from the algebraic argument: the second intersection is a
`K_S^ℂ`-rational point and is reachable by field operations.

### 6.6 Completing the sufficiency proof

**Theorem (compass-only sufficiency)**. For every finite starting set
`S` with `|S| ≥ 2`, `Reach({C}, S) = Reach({L, C}, S)`.

**Proof**. By §5 (generalized to `ℂ` at §6.2),
`Reach({L, C}, S) = {K_S^ℂ\text{-rational points}}`. By §6.3 and
§6.4, every complex-arithmetic generator (`+, −, ·, /, √`) has a
compass-only realization on constructed points. Any `K_S^ℂ`-element
is a finite composition of these generators starting from the
`S`-coordinates; by finite composition of compass-only
constructions, every `K_S^ℂ`-rational point is compass-only
constructible. So `Reach({C}, S) ⊇ K_S^ℂ\text{-rational points} =
Reach({L, C}, S)`. The reverse inclusion
`Reach({C}, S) ⊆ Reach({L, C}, S)` is immediate since every
`C`-primitive is also an `{L, C}`-primitive. Equality follows. ∎

## 7. The line primitive alone is not universally sufficient

**Claim**: There exists a starting set `S` such that
`Reach({L}, S) ⊊ Reach({L, C}, S)`.

**Proof**. Take `S = {A, B}` with `A ≠ B`. Using only `L`:

- The only pair of distinct constructed points available initially is
  `(A, B)`. Applying `L` gives `line(A, B)`. This line has no prior
  curves to intersect with, so no new points are produced.
- After the first step, we still have only `{A, B}` as constructed
  points, and only `line(A, B)` as a drawn curve.
- Applying `L` again requires two distinct constructed points; the
  only pair available is still `(A, B)`. This produces the same
  `line(A, B)` again — no new curve, no new points.

So `Reach({L}, {A, B}) = {A, B}`.

Meanwhile, `Reach({L, C}, {A, B})` contains at least `A, B`, the two
equilateral-triangle vertices `E_1, E_2 =` intersections of
`circle(A, B)` and `circle(B, A)`, their reflections, and so on — an
unbounded set including the midpoint `M_{AB} = (A+B)/2`
(constructible by dropping the perpendicular bisector of `AB` via
two `C`-steps and one `L`-step).

`M_{AB} ∉ Reach({L}, {A, B}) = {A, B}`, but
`M_{AB} ∈ Reach({L, C}, {A, B})`. Hence
`Reach({L}, {A, B}) ⊊ Reach({L, C}, {A, B})`. ∎

**Strengthening**. The same argument generalizes: for any `S` whose
projective closure equals `S` itself (for instance, `S` collinear, or
`S` in "general position" of `≤ 3` points), `{L}` alone cannot
construct any non-trivial auxiliary point, but `{L, C}` can. The
collinear / two-point case is the cleanest counterexample.

(The `{L}`-reachable set in general is the "projective completion" of
`S`: all points expressible as projective combinations of `S`
involving only incidences and line-line joins. This set is strictly
smaller than the quadratic closure `K_S`-rational set in general,
because projective combinations cannot introduce square roots. For
`|S| ≥ 4` in sufficiently general position, `{L}` does reach many
points, but never any point whose coordinates require an irrational
square root — so e.g. `{L}` cannot construct `(A+B)/2` from generic
starting sets unless midpoint information is already encoded
projectively in `S`.)

## 8. The minimal sufficient subset

We have:
- `∅` is not sufficient (reaches only `S`).
- `{L}` is not universally sufficient (§7).
- `{C}` *is* universally sufficient (§6).
- `{L, C}` is the full set, trivially sufficient.

Thus the smallest subset `T ⊆ {L, C}` that is universally sufficient is
**`T = {C}`**.

`{C}` is a singleton, and no proper subset of it (only `∅`) is
sufficient. So `{C}` is also irreducible: it is the *unique*
minimal universally sufficient subset.

## 9. Verification strategy

How would one confirm that a candidate primitive set is universally
sufficient? The argument above establishes sufficiency for `{C}` via a
*proof*, not an empirical test. Confirming in general has two
complementary approaches:

### 9.1 Algebraic verification

Compute the invariant `Reach(T, S)` symbolically — that is, identify
the algebraic closure of the coordinate-field of `S` under the
operations that `T` permits, and compare it to `K_S^ℂ`.

For `{C}`:
- Each `C`-step (given compass-only inversion, reflection, midpoint,
  doubling) can realize arbitrary field operations and square roots
  on coordinates.
- Every `K_S^ℂ`-element arises from a finite expression in these
  operations.
- Hence the invariant `Reach({C}, S)` equals
  `{K_S^ℂ-rational points}`, the same invariant as
  `Reach({L, C}, S)`.

For an arbitrary candidate `T`, the procedure is:
1. For each primitive in `T`, determine the maximal algebraic
   extension it can induce in one step.
2. Close the coordinate field under these induced extensions; call the
   closure `K_T(S)`.
3. `T` is universally sufficient iff `K_T(S) = K_S^ℂ` for every `S`.

### 9.2 Structural-induction verification

Verify by induction on construction length: every point a
`{L, C}`-construction can produce in `n` steps is also produced by
some `T`-construction in `f(n)` steps (for some function `f`).

Induction hypothesis: every point constructed in `≤ n` `{L, C}`-steps
is in `Reach(T, S)`.

Induction step: a point constructed in `n+1` steps arises as the
intersection of two curves. If the final step's curve is a `T`-legal
primitive, the step is directly reproducible. Otherwise, simulate
the dropped primitive by `T`-sub-sequences — using the coordinate-
level field operations of §6.3 as the simulators.

The two verification strategies are equivalent in content; the first
is cleaner (a statement about fields), the second is closer to the
operational definition (a statement about constructions).

### 9.3 Falsification test (for a candidate `T`)

To falsify universal sufficiency: exhibit a starting set `S_0` and a
point `X_0 ∈ Reach({L, C}, S_0) \ Reach(T, S_0)`.

For `T = {L}`, the falsifier is `S_0 = {A, B}`, `X_0 =` midpoint
of `AB` (§7).

For `T = ∅`, the falsifier is any `S_0 = {A, B}` and
`X_0 =` any point not in `S_0` — such as an equilateral-vertex on
`AB`.

For `T = {C}`, no falsifier exists (Theorem 6.6).

## 10. Worked examples

Each example uses only `{C}` — compass operations (draw a circle
centered at one already-constructed point passing through another) and
the extraction of their intersection points with already-drawn
circles.

### 10.1 Midpoint of a segment `AB`

**Input**: Points `A, B` with `A ≠ B`.

**Output**: `M = (A + B) / 2`.

**Construction** (from Lemma 4.4):

1. Draw `K_1 = circle(B, A)` (center `B`, through `A`, radius
   `r = |AB|`).
2. Draw `K_2 = circle(A, B)` (center `A`, through `B`, radius `r`).
   Intersections of `K_1` and `K_2`: two points, call them `X_1, X_2`,
   forming equilateral triangles with `A` and `B`.
3. Draw `K_3 = circle(X_1, B)`. Its intersections with `K_1`: two
   points; one is `A`, the other is a new point `Y_3` on `K_1` at
   angular displacement `120°` from `A` (measured from `B`).
   Specifically, placing `A = (0,0), B = (r, 0)`, we have
   `X_1 = (r/2, r√3/2)` and `Y_3 = (3r/2, r√3/2)` (the label `Y_3`
   is used here, distinct from the `X_3` of Lemma 4.2, to avoid
   collision since `Y_3` and Lemma 4.2's `X_3` sit in different
   coordinate frames).

   Coordinate verification: `K_1` is `(x - r)² + y² = r²` and
   `K_3 = circle(X_1, B)` is centered at `(r/2, r√3/2)` with radius
   `|X_1 B| = √((r/2)² + (r√3/2)²) = r`. Intersection of `K_1` and
   `K_3`:
   `(x - r)² + y² = r²`,
   `(x - r/2)² + (y - r√3/2)² = r²`.
   Subtract: `(x - r)² - (x - r/2)² + y² - (y - r√3/2)² = 0`, which
   simplifies to `-rx + r√3 y = 0`, i.e. `y = x/√3`.
   Substitute into `K_1`: `(x - r)² + x²/3 = r²`, so
   `(4/3) x² - 2rx = 0`, giving `x = 0` (point
   `A = (0,0)`) or `x = 3r/2` (point `Y_3 = (3r/2, r√3/2)`). ✓

4. Draw `K_4 = circle(Y_3, B)`. Its intersections with `K_1`: two
   points; one is `X_1`, the other is a new point `Y_5` on `K_1` at
   angular displacement `240°` from `A` (= `60°` before `2π`). By
   symmetry (or direct computation), `Y_5 = (2r, 0)`. That is,
   `Y_5 = 2B - A =` antipode of `A` on `K_1`. Call it `B̃ = 2B - A`.
5. Now invert `B̃` in the circle `ω = circle(A, B)` of radius `r`.
   By Lemma 4.3, Case 1 (since `|A B̃| = 2r ≥ r/2`):
   a. Draw `K_5' = circle(B̃, A)` (radius `|B̃ A| = 2r`).
   b. `K_5'` intersects `ω` at two points `C_1, C_2` with
      `|A C_i| = r` and `|B̃ C_i| = 2r`.
   c. Draw `K_6 = circle(C_1, A)` (radius `r`, center `C_1` on
      `ω`) and `K_7 = circle(C_2, A)` (radius `r`).
   d. `K_6` and `K_7` intersect at `A` and at a second point, which is
      the inverse `B̃^*`. By Lemma 4.3, this is the midpoint `M =
      (A + B)/2`.

**Verification**: With `A = (0, 0)`, `B = (r, 0)`, we have
`B̃ = (2r, 0)`. Inversion in `circle(A, r)` sends
`B̃ = (2r, 0)` to `(r² / (2r), 0) = (r/2, 0) = (A + B)/2 = M`. ✓

**Step count**: `K_1, K_2, K_3, K_4, K_5', K_6, K_7` = 7
compass-circle primitives, plus intersection extractions (free per
step). The construction is finite and deterministic (given consistent
choices of "the new intersection" at each step).

### 10.2 Foot of perpendicular from a point `P` to line `AB`

**Input**: Distinct points `A, B` (defining `ℓ = line(A, B)`, not
drawn) and a point `P`.

**Output**: `F =` foot of perpendicular from `P` to `ℓ`.

**Construction** (from Lemmas 4.1 and 4.4):

1. By Lemma 4.1, reflect `P` across `ℓ`:
   a. Draw `J_1 = circle(A, P)` (center `A`, through `P`, radius
      `|AP|`).
   b. Draw `J_2 = circle(B, P)` (center `B`, through `P`, radius
      `|BP|`).
   c. Intersections of `J_1, J_2`: two points; one is `P`, the other
      is `P'` — the reflection of `P` across `ℓ`.
2. By Lemma 4.4, construct the midpoint `F = (P + P') / 2`. (This is
   seven more compass-circle primitives as in §10.1.)

**Verification**: Reflection `P'` satisfies `|P'A| = |PA|` and
`|P'B| = |PB|`, so `P'` is the unique point other than `P` equidistant
from both `A` and `B` (by the matching radii). Geometrically: `P'` is
the reflection of `P` across the perpendicular bisector of every
segment `PP'` passing through the line `AB`, which means `PP'` is
perpendicular to `AB` and the midpoint of `PP'` lies on `AB`.
Therefore `F = (P + P')/2` is on `AB` (i.e., on `ℓ`), and `PF ⊥ ℓ` —
exactly the foot of the perpendicular. ✓

**Alternative verification by coordinates**: Place `A = (0, 0)`,
`B = (b, 0)` (so `ℓ = x`-axis), `P = (p, q)` with `q ≠ 0`. Then
`|AP|² = p² + q²`, `|BP|² = (p-b)² + q²`. Reflection of `P` across
`x`-axis is `P' = (p, -q)`, and `|AP'|² = p² + q² = |AP|²` ✓,
`|BP'|² = (p-b)² + q² = |BP|²` ✓. So `P' = (p, -q)` is indeed the
other intersection of the two circles `J_1, J_2`. Midpoint
`(P + P')/2 = (p, 0)`, which is on `ℓ` at horizontal coordinate `p`
— the foot of the perpendicular from `P = (p, q)` to the `x`-axis. ✓

**Step count**: 2 (reflection) + 7 (midpoint) = 9 compass primitives.
Can be tightened by sharing intermediate circles, but 9 is an upper
bound.

**Degenerate case**: If `P ∈ ℓ`, the two circles `J_1` and `J_2` are
tangent at `P`, and `P' = P`, so `F = P`. The construction correctly
returns `P`.

### 10.3 Reflection of a point `P` across line `AB`

This is Lemma 4.1 directly.

**Input**: Distinct `A, B` and a point `P`.

**Output**: `P' =` reflection of `P` across `line(A, B)`.

**Construction**:

1. Draw `J_1 = circle(A, P)`.
2. Draw `J_2 = circle(B, P)`.
3. Intersect: two points. One is `P`, the other is `P'`.

**Verification**: `|AP'| = |AP|` (both are radii of `J_1`) and
`|BP'| = |BP|` (both are radii of `J_2`). This means `P, P'` are
equidistant from `A` and from `B`. The locus of points equidistant
from `A` is a circle around `A`; similarly for `B`. So `P, P'` lie on
both loci. The two loci intersect in two points (generically), and `P`
is one of them. `P'` is the other.

Geometrically: the perpendicular bisector of segment `P P'` passes
through both `A` and `B` (by equal distances), so it is the line `AB`.
Therefore `P'` is the mirror image of `P` across `AB`. ✓

**Step count**: 2 compass primitives (plus intersection extraction,
free).

### Summary: all three use only the `C` primitive

Each of §10.1, §10.2, §10.3 involves only:
- Drawing circles centered at already-constructed points and passing
  through already-constructed points (the `C` primitive).
- Taking the intersection points of these circles (the secondary
  output of `C`).
- No use of the `L` primitive anywhere.

All three are therefore witnesses that the three named canonical
constructions are `{C}`-reachable, consistent with §6.

## 11. Open questions and limitations

### 11.1 Ambiguities in "every point constructible"

The sufficiency theorem (§6.6) says every *point* reachable by
`{L, C}` is reachable by `{C}`. It does not say every *drawn curve*.
Under `{C}`, no line is ever drawn as a curve. For applications that
care about the sequence of drawn curves (e.g., proving a straightedge
construction exists that produces a specific drawing), our result is
insufficient; but for the task as stated — which is phrased in terms
of constructible points — it is complete.

### 11.2 Complexity cost

Compass-only simulation is polynomial-step per `{L, C}`-step but with
a multiplicative overhead. Specifically:

- A line-circle intersection costs ~4 compass steps (2 reflections + 1
  circle + 1 intersection extraction).
- A line-line intersection via the inversion route costs many more:
  ~4 inversions + ~6 reflections + the second-intersection
  sub-construction (~10+ steps). Exact counts depend on the specific
  sub-routine chosen.

A careful accounting would give `f(n) = O(n)` with a large constant.
Empirically tight bounds (minimum compass steps to simulate a given
`{L, C}`-construction) are an unresolved optimization question.

### 11.3 Extension to non-Euclidean geometries

In hyperbolic and spherical geometries, the "line" and "circle"
primitives have counterparts (geodesic through two points, metric
ball around a point through a second point). The analogous question —
does the compass alone suffice — has a different answer in different
curvatures. We have not addressed these cases; the field-theoretic
argument in §5–§6 relies on the Euclidean coordinate system.

### 11.4 Smaller than `{C}`?

The compass primitive is a binary operation. Could one imagine a
"sub-compass" primitive that is also sufficient? For instance:

- "Compass with fixed radius" (the rusty compass): can only draw
  circles of a single pre-fixed radius. This is strictly weaker than
  full `{C}`: many compass-only constructions require varying radii.
- "Circle through three given points" (a 3-ary primitive): reducible
  to pairwise compass operations, not obviously stronger or weaker.

These are distinct primitive sets from `{L, C}`, so fall outside the
scope of our question. A careful exploration of primitive-set
equivalence classes is open.

### 11.5 Degenerate starting configurations

§7's claim that `{L}` is not universally sufficient rests on starting
sets whose projective closure is trivial (e.g., two points). For
richer `S` — in particular, `S` already containing a circle with its
center — classical results in the adjacent direction show `{L}` alone
is sufficient. Our result does not make a direct claim in this
regime; we only claim non-universality. A full characterization of
"which `S` make `{L}` alone sufficient" is a classical question we
have not answered.

### 11.6 Constructability of `√` in place of `C`

Our sufficiency argument factors through: field operations are `{C}`-
realizable, square root is `{C}`-realizable. An alternative primitive
set would be one that realizes `(+, ×, √)` in a single step each —
this would obviously be sufficient, but it does not live inside
`{L, C}` and is therefore not a subset in our sense. Exploring
minimal primitive sets drawn from richer universes is open.

### 11.7 Number of degrees of freedom

The compass primitive has 2 degrees of input (two points). The line
primitive also has 2. In this strict input-counting sense, neither is
"smaller" than the other. Our minimality result is about universal
simulation power, not input dimensionality.

### 11.8 "Starting from `S`" vs "starting from `∅`"

If `S = ∅` (no starting points), neither `{L}` nor `{C}` can construct
anything; universality is vacuous. If `S` has a single point, both are
still trivial. The minimal interesting case is `|S| ≥ 2`, which is
what we assume throughout.
